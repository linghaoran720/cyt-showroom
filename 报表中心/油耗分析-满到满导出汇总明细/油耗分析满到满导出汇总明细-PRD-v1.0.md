# 油耗分析满到满导出（长周期汇总 + 去重加油明细）

## 一、产品概述

### 1.1 产品定义

在**油耗分析 → 满到满报表**中，在现有「导出全部 / 导出选中」基础上，**新增**一类导出：对客户勾选的满到满记录生成 **Excel**，且 **仅包含两块内容**——**长周期汇总区**、**去重加油明细区**。

- **长周期汇总区**：**仅按车牌号** 聚合。每个车牌在本次导出中对应 **一行**汇总（多 Tab 时每 Sheet 一行），将客户勾选的、属于该车牌的全部满到满段按 **§5.3** 规则合成一条长周期指标；**不**再导出与现网「导出选中」相同的 **逐段满到满原始行**（避免与线上下载主表重复）。
- **去重加油明细区**：汇总区下方第二套表头；覆盖本次勾选涉及的全部加油记录，按 **在途费用编号** 去重，解决相邻满到满段边界加油重复问题。
- **不包含**：同车不同司机的 **分行汇总** 规则；不按司机拆分长周期汇总。

表头字段集合仍可按 **§5.3** 与现网满到满列表（分油耗模式）**对齐列名与口径**，便于业务理解；语义上是 **长周期专用汇总行**，而非线上下载那份「一段一行」的主表复制品。

### 1.2 产品目标

- **一次导出**仅得到：**按车牌聚合的长周期汇总（每车牌一行）** + **全量去重加油明细**；与「导出选中」线上下载的满到满逐行表 **不重复提供**。
- 版式参考「车辆运营报表 → 导出选中运单明细」：**同一 Sheet 内两个表头区块**（长周期汇总表 + 加油明细表）。

### 1.3 核心价值

| 维度 | 价值 |
|------|------|
| 效率 | 少步骤、少文件；无需为长周期再手工合并线上下载 + 抽屉明细 |
| 数据质量 | 明细按在途费用编号去重，避免段间边界重复 |
| 口径清晰 | 长周期只看车牌维度汇总，规则简单、实现与验收成本低 |

## 二、用户分析

### 2.1 主要用户角色

| 角色 | 职责 | 使用场景 |
|------|------|----------|
| 统计员 / 运营 | 核对油耗、对账、复盘 | 勾选多段满到满，导出汇总与明细 |

### 2.2 用户痛点

1. 仅「导出选中」时无 **按车牌** 的长周期聚合，难以对应「一车跨多段」管理视角。
2. 明细需在「查看明细」中再导出，且相邻段边界加油记录重复，需手工去重。

## 三、核心功能

### 3.1 功能清单

| 功能 | 描述 |
|------|------|
| 导出入口 | 在满到满列表导出菜单中新增选项：导出选中明细 |
| 按车牌分 Sheet | 每个车牌号对应 Excel 一个Sheet；Sheet 命名需处理非法字符与长度 |
| 长周期汇总区 | **仅按车牌号** 聚合（见 5.2）；每车牌 **一行**。列名与口径可参考现网满到满列表（§5.3，分油耗模式），各列聚合方式由产品定稿；**不**附带逐段满到满原始行 |
| 明细区 | 汇总区下方第二套表头：本次勾选涉及的全部加油明细；按 **在途费用编号** 去重；**不按司机分块** |
| 兼容性 | 保留原有「导出选中」仅主表行为，不因本需求改变 |
| 勾选上限 | 本导出：**单次最多勾选 500 条**满到满主表行；超出则提示并禁止导出（性能与稳定性约束） |

### 3.2 分期与降级（成本可控）

- **优先**：多车牌导出 → 多 Sheet + 每 Sheet 内「汇总 + 明细」。
- **可接受降级**：若实现成本过高，**M1 限制为仅允许勾选同一车牌**；校验失败时提示用户，不发起导出。

## 四、用户场景

### 场景 1：同车牌多段满到满 → 一行长周期汇总 + 去重明细

**用户**：统计员  
**目标**：将 **同一车牌** 下勾选的多段满到满视为一长周期，并导出全部加油笔数且每笔只出现一次。

**流程**：

1. 在满到满列表勾选多行（须为 **同一车牌**，若启用 M1 多车牌限制则分次导出）。
2. 选择「长周期汇总 + 加油明细」类导出入口。
3. 打开 Excel：该车牌所在 Sheet **汇总区仅一行**（该车牌下勾选各段聚合结果）；**明细区**为去重后的加油列表。

**预期结果**：每车牌 Sheet 内汇总 **1 行**；明细无重复在途费用编号；加油时间整体升序（同秒顺序不限）。

### 场景 2：多车牌（若本期支持多 Tab）

**用户**：统计员  
**目标**：一次勾选多车（在条数上限内），每车各得长周期汇总与明细。

**预期结果**：**每个车牌一个 Sheet**；各 Sheet 内仍为 **汇总1 行 + 明细表**；明细仅含该车牌本次勾选段涉及的费用（去重规则不变）。

**说明**：**不按司机**拆分汇总。若用户勾选同牌不同司机的多段，仍合并为该车牌 **一行**长周期汇总；**司机姓名**列对涉及的多名司机 **去重后** 用 **英文逗号 `,`** 拼接（拼接顺序等产品在 **§5.3 聚合说明表** 中注明）。

## 五、功能详细说明

### 5.1 导出与校验

**功能描述**：新增导出类型；可选 M1 校验「仅单车牌」。

**校验规则**：

- **勾选条数**：满到满主表勾选行数 **> 500** → 提示（文案需说明上限为 500 条），**不发起导出**。
- 若启用降级策略：勾选行车牌号不一致 → 提示，不导出。
- 未勾选任何行 → 提示，不导出。
- **本需求不校验**「同车必须同司机」：允许同车牌、不同司机的段被一并勾选，仍按 **车牌** 合成一行（司机列口径见 §5.3）。

### 5.2 长周期汇总区：仅按车牌号聚合

**分组键**：**仅车牌号**（不按司机拆分）。

| 勾选情况 | 汇总区行数（单 Sheet 内） |
|----------|---------------------------|
| 同一车牌、勾选 N 段 | **始终1 行**（N 段指标按 §5.3 聚合） |
| 多车牌且本期支持多 Sheet | 每车牌独立 Sheet，各 **1 行** |

**与线上下载关系**：现网「导出选中」仍导出 **逐段**满到满行；**本导出**汇总区 **不重复**输出这些原始段行，只输出长周期聚合后的 **一行**。

### 5.3 汇总字段聚合方式（待产品确认）

以下字段来自客户提供的 **满到满列表全表导出 Excel**（与线上列表列一致）。因 **油耗模式** 不同，表结构分为两套；**长周期汇总表**选用与当前模式对应的一套列，聚合对象为：**同一车牌号** 下勾选的全部满到满行（见 5.2）。

**说明**：

- 「建议聚合方式」为便于评审的初稿，**请你直接改表内文字定稿**。
- 符号约定：**最小**、**最大**、**分组展示**（与 **车牌号** 分组一致；车牌列组内恒相同）、**按汇总重算**、**待确认**。
- **司机姓名**：勾选段涉及多名司机时，**去重后** 用 **英文逗号 `,`** 连接各司机姓名（**拼接顺序**由产品在 §5.3 聚合说明表中注明，可与研发约定如按姓名升序等）。
- **车管、车辆所属组织** 等：若跨段不一致，规则由产品在 **聚合说明表** 中另行注明（本 PRD 不强制与司机列相同处理方式）。

### 5.4 明细区：去重、排序、字段

**去重键**：**在途费用编号**（系统字段；界面若未展示，导出仍应用于去重逻辑）。

**去重规则**：对本次导出范围内、所有勾选满到满段关联的加油明细做并集，再按在途费用编号 **distinct**；同一编号只保留一条。

**排序**：按 **加油时间** 降序；**加油时间相同** 的记录之间 **顺序无要求**（实现上可为任意顺序；不要求稳定排序）。

**结构**：明细区为 **一张表**，列与「查看明细 → 加油明细」对齐；**不按司机分组展示**。

### 5.5 与原有导出关系

- 「导出选中 / 导出全部」：仍为 **逐段**满到满主表，**不包含**本需求的长周期汇总逻辑与 **去重明细区块**。
- **本新增导出**：**不包含**逐段满到满原始行，避免与线上下载主表 **内容重复**；需要段级明细时用户继续使用原「导出选中」。
- 抽屉内「加油明细」导出（如有）：保留与否由研发评估；本需求以列表侧一键导出为主路径。

## 六、验收标准

- [ ] 勾选 **501 条及以上** 时：提示清晰，**不生成文件**；**500 条及以下**可正常导出。
- [ ] **同一车牌**勾选 N 段：该车牌 Sheet 内汇总区 **仅 1 行**（长周期聚合），**不出现** N 行逐段满到满主表。
- [ ] 明细区按在途费用编号 **无重复**；按加油时间升序，同秒顺序任意均可接受。
- [ ] 多车牌时（若本期实现）：**每车牌独立 Sheet**，各 Sheet 汇总 **1 行** + 明细表。
- [ ] 降级 M1（若采用）：多车牌勾选时提示清晰，不生成文件。
- [ ] 原有「导出选中」仍为逐段主表，**不因本需求改为长周期或夹带本明细区块**。
- [ ] 同车牌、不同司机一并勾选时：汇总仍为 **1 行**，**司机姓名** 为去重后 **英文逗号 `,`** 拼接，顺序符合 §5.3 聚合说明表。

## 七、风险与说明

| 风险项 | 说明 |
|--------|------|
| 汇总与明细金额/升数口径 | 需在实现前确认：汇总列是否必须与明细求和一致，或以服务端汇总逻辑为准 |
| 姓名内含逗号 | 司机名本身若含英文逗号，可能与分隔符歧义；概率极低，若出现由客服/数据规范约束或后续迭代转义 |
| 性能与数据量 | **单次导出最多勾选 500 条**满到满主表行；降低服务端与浏览器压力。明细行数随勾选段数增长，仍可能较大；若未来用户频繁顶格使用仍慢，再评估异步导出等方案（本期不展开） |
