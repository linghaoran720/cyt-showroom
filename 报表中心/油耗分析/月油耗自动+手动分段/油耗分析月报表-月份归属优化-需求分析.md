# 需求分析报告

## 1. 需求概述
- **需求名称**：油耗分析月报表 - 满到满周期月份归属逻辑优化
- **需求类型**：功能优化
- **需求来源**：客户业务场景反馈
- **优先级建议**：P1高

## 2. 业务背景

### 当前现状

司机根据加油情况登记加油信息，加满油时标记"已加满"。系统根据加满记录进行分段，切分成最小单元「满→满」生成满到满报表。部分客户按月份管理油耗时，系统在「满→满」基础上，按固定逻辑合并为月度报表：**上月最后一次加满 → 本月最后一次加满 = 本月油耗**。若上月无加满记录则往前找3个月作为开始；若本月无加满记录则本月记录为空。

### 客户痛点

客户会管理司机加油行为，正常情况下每月月初/月底会有一次加满（背景：公司采购油价便宜，且不加满可以多装货）。但实际情况中部分车辆因任务无法在月底回来，只能月初回来补满，导致：

1. **一部分车月底加满**：上月底→本月底，计算相对合理
2. **一部分车月初加满**：本月初→下月初，系统按固定逻辑会把两个月油耗合并到一个月
3. **一部分车跨月**：上月初→下月初，本月完全无加满记录，系统显示本月无数据

### 期望目标

- 优化满到满的周期节点选取逻辑，自动适配"月底/月初加满"的不同节奏
- 减少因固定逻辑导致的空月和数据堆积
- 提供人工修正能力作为兜底

### 业务价值

- 油耗数据更准确，减少空月和堆积
- 奖惩更公平，减少司机争议
- 减少人工干预量，降低管理成本

## 3. 用户角色与场景

| 角色 | 场景描述 | 核心诉求 |
|------|----------|----------|
| 车队管理员 | 每月查看油耗月报表，评估司机表现 | 每月都有合理的油耗数据，减少空月或堆积 |
| 财务人员 | 根据月报表计算油耗奖惩金额 | 奖惩金额准确归属到对应月份 |
| 司机 | 按公司要求在月底/月初加满油 | 油耗奖惩公平合理 |
| 系统管理员 | 对异常数据进行人工修正 | 有便捷的修正工具，修正后自动重新计算 |

## 4. 功能需求拆解

### 4.1 核心功能

#### F1：优化周期节点自动选取逻辑

##### 设计思路 - 三层机制

| 层级 | 机制 | 说明 |
|------|------|------|
| 第一层 | 系统自动计算（优化后） | 覆盖大部分标准及常见场景，减少人工介入 |
| 第二层 | 人工修正 | 对自动计算不合理的情况，允许手动调整周期节点 |
| 第三层 | 业务约束前提 | 客户需管理司机加满行为，确保合理时机加满 |

##### 自动计算逻辑（第一层）

计算N月油耗时，需确定"开始节点"和"结束节点"：

**找开始节点（按优先级）**：

| 优先级 | 条件 | 开始节点 |
|--------|------|----------|
| 1（最高） | N-1月有加满记录 | N-1月**最后一次**加满 |
| 2 | N-1月无加满，N月有加满 | N月**第一次**加满（同时作为N-1月油耗的结束节点） |
| 3 | N-1月无加满，N月也无加满 | N月无油耗数据 |

**找结束节点（按优先级）**：

| 优先级 | 条件 | 结束节点 |
|--------|------|----------|
| 1（最高） | N月有加满记录 | N月**最后一次**加满 |
| 2 | N月无加满，N+1月有加满 | N+1月**第一次**加满 |
| 3 | N月无加满，N+1月也无加满 | N月无油耗数据 |

**N月油耗 = 开始节点 → 结束节点**，周期内所有加油记录（含非满油）整体归属到N月。

##### 场景验证（计算2月油耗）

| 场景 | 1月加满 | 2月加满 | 3月加满 | 开始节点 | 结束节点 | 2月油耗周期 |
|------|---------|---------|---------|----------|----------|------------|
| 标准场景 | 月底有满 | 月底有满 | - | 1月最后满（优先级1） | 2月最后满（优先级1） | 1月底→2月底 |
| 月初加满 | 无满 | 月初+月底各有满 | - | 2月第一次满（优先级2） | 2月最后满（优先级1） | 2月初→2月底 |
| 仅月初满 | 无满 | 月初有满 | 月初有满 | 2月第一次满（优先级2） | 3月第一次满（优先级2） | 2月初→3月初 |
| 本月无满 | 月底有满 | 无满 | 月初有满 | 1月最后满（优先级1） | 3月第一次满（优先级2） | 1月底→3月初 |
| 前后都无满 | 无满 | 无满 | 无满 | 无法确定 | 无法确定 | 无数据 |

##### 关键约束

- 每次加满既是上一周期的结束，也是下一周期的开始，周期无缝衔接
- 同一加油记录只归属一个周期，不会重复计算
- 月内有多次加满时，最后一次为绿色节点（周期切割点），其余降为红色（仅参与加油量计算）
- 月内仅有一次加满时，该次加满既可能是开始节点也可能是结束节点，由上下文决定
- 车辆在系统内最早的加油记录默认视为满油，作为首个周期的起始节点

#### F2：人工修正能力（第二层）

##### 适用场景

| 场景 | 说明 | 操作方式 |
|------|------|----------|
| 有计算结果但不合理 | 当月有满到满周期，但开始/结束时间不符合客户管理要求 | 直接修改满到满的开始/结束时间 |
| 无计算结果 | 当月无满到满周期（无加满记录） | 先修改加油记录标记为加满，或单独添加一条加满记录，刷新后再修正 |

##### 修正交互流程

1. 管理员在月报表中勾选**某月某车**，点击"修改"
2. 系统展示当前满到满周期的**开始时间**和**结束时间**
3. 管理员通过下拉选择新的开始时间和结束时间
4. 提交保存

##### 可选时间范围

下拉可选的时间节点需同时满足：
- **开始时间可选范围**：上月满到满开始时间**之后** ~ 当月满到满结束时间**之前**，且为标记"加满"的加油记录
- **结束时间可选范围**：当月满到满开始时间**之后** ~ 下月满到满结束时间**之前**，且为标记"加满"的加油记录

> 示例：修改2月，1月周期为12月31日→1月30日，3月周期为2月28日→3月31日
> - 开始时间可选：12月31日之后 ~ 2月28日之前的所有加满记录
> - 结束时间可选：选定的开始时间之后 ~ 3月31日之前的所有加满记录

##### 提交校验

- 开始时间必须早于结束时间

##### 保存后影响

- 系统自动联动更新**前一月、当月、后一月**共三个月的满到满时间节点及油耗数据
- 因可选范围被上月开始时间和下月结束时间约束，修改影响范围最多三个月，不会继续扩散

#### F3：周期内所有加油记录纳入计算

满到满周期内，所有加油记录的加油量均纳入该周期"实际升数"计算：

| 记录类型 | 周期切割作用 | 纳入加油量计算 |
|----------|-------------|---------------|
| 绿色满油记录（周期节点） | 是 | 是 |
| 红色满油记录（非节点） | 否 | 是 |
| 非满油加油记录 | 否 | 是 |

#### F4：特殊场景 - 司机离职结算

司机离职时进行一次加满操作，该加满记录作为当前周期的结束点，触发油耗结算。

### 4.2 辅助功能

#### F5：历史数据刷新

- 系统已有手动刷新功能，需确保新逻辑在刷新时生效
- 客户可选择指定月份重新计算

### 4.3 非功能需求

- **数据一致性**：同一加油记录不会同时归属到不同月份周期，避免重复计算
- **性能要求**：历史数据批量刷新不影响系统正常使用

## 5. 边界与约束

### 包含（In Scope）
- 周期节点自动选取逻辑优化
- 人工修正周期节点能力
- 修正后自动重新计算
- 非满油加油记录纳入周期计算
- 司机离职结算场景
- 历史数据刷新兼容新逻辑
- 车辆首条加油记录默认视为满油

### 不包含（Out of Scope）
- 分段明细的分段逻辑不变（空重驶、国道高速等影响因素）
- 标准百公里油耗配置不变
- 奖惩系数配置不变
- 满到满的基本计算模式不变（仍以加满记录为周期节点）

### 业务约束前提
- 客户使用此功能需遵循满→满的基本原则
- 客户需对司机加满油行为有一定约束（如要求在月底/月初加满）
- 完全无序的加满行为，系统和人工都无法合理解决

### 依赖项
- 现有满油记录数据（`tank_full = true`）
- 现有在途费用数据（含非满油加油记录）
- 现有分段明细计算引擎

## 6. 风险评估

| 风险项 | 影响程度 | 应对策略 |
|--------|----------|----------|
| 新旧逻辑切换导致历史数据不一致 | 高 | 提供历史数据刷新，客户按需操作 |
| 自动逻辑仍无法覆盖所有场景 | 中 | 人工修正作为兜底，覆盖极端情况 |
| 人工修正操作不当导致数据异常 | 中 | 修正前展示影响范围预览，支持撤销操作 |
| 跨多月无数据，客户误解为异常 | 低 | 报表中对无数据月份给出说明提示 |

## 7. 验收标准

### 自动计算逻辑
- [ ] 标准场景：1月底满、2月底满 → 2月油耗 = 1月底满→2月底满
- [ ] 月初加满：1月无满、2月初满+2月底满 → 2月油耗 = 2月初满→2月底满
- [ ] 仅月初满：1月无满、2月初满、3月初满 → 2月油耗 = 2月初满→3月初满
- [ ] 本月无满：1月底满、2月无满、3月初满 → 2月油耗 = 1月底满→3月初满
- [ ] 前后都无满：1月无满、2月无满、3月无满 → 2月无数据
- [ ] 车辆首条加油记录默认视为满油，正确作为首个周期起点

### 人工修正
- [ ] 有计算结果时，可勾选某月某车点击修改，修改满到满的开始/结束时间
- [ ] 无计算结果时，不支持直接修改，需先添加/标记加满记录后刷新
- [ ] 下拉可选时间范围正确：开始时间 > 上月周期开始，结束时间 < 下月周期结束，且均为加满记录
- [ ] 提交时校验开始时间早于结束时间
- [ ] 保存后自动联动更新前一月、当月、后一月的数据
- [ ] 修改影响范围不超过三个月

### 数据完整性
- [ ] 周期内所有加油记录（含非满油）正确纳入实际升数
- [ ] 同一加油记录不会出现在不同月份周期中
- [ ] 奖惩金额按新逻辑正确计算到对应月份

### 其他
- [ ] 司机离职加满后正确触发结算
- [ ] 手动刷新历史月份时使用新逻辑
