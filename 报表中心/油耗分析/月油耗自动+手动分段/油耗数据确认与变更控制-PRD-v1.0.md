## 1. 概述

### 1.1 项目背景

财运通油耗分析模块中，司机新增或修改油费上报时，系统会自动更新油耗分析数据。如果新增或修改的油费是加满记录，可能影响满到满的分段周期，导致财务人员之前看到的月报表数据产生变化。

当前系统没有数据确认/锁定机制，任何油费变更都会直接影响报表数据，存在以下问题：
- 财务已基于报表数据完成奖惩核算，但司机后续补录/修改导致数据变化
- 数据变化时财务人员无感知，可能导致错误的奖惩发放

> 说明：油费上报本身已有独立的审批流程，本需求不涉及油费审批，仅控制油费变更对油耗分析报表的影响。

### 1.2 项目目标

- 建立月度油耗数据的"确认"机制，区分已确认和未确认状态
- 未确认数据允许自动变更，适应司机正常上报节奏
- 已确认数据变更时提醒管理员，由管理员确认后更新报表
- 变更影响满到满周期时，明确提示影响范围

### 1.3 项目范围

**包含（In Scope）**
- 月度数据确认/撤回确认机制
- 未确认数据自动变更
- 已确认数据变更提醒与确认更新流程
- 变更影响检测与提醒
- 满到满周期变更影响提示
- 变更记录查询

**不包含（Out of Scope）**
- 油费上报的审批流程（已有独立流程）
- 按车辆维度的部分确认（本次按月份整体确认）
- 到期自动确认

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| 数据确认 | 财务人员对某月油耗报表数据执行确认操作，锁定数据不被自动更新 |
| 变更提醒 | 司机油费变更影响已确认月份时，系统自动生成的提醒通知 |
| 确认更新 | 管理员查看变更影响后，确认将变更同步到报表数据 |
| 变更检测 | 系统自动判断油费变更是否影响已确认月份数据的检测逻辑 |

## 3. 需求详述

### 3.1 功能清单

| 编号 | 功能名称 | 优先级 | 描述 |
|------|----------|--------|------|
| F001 | 月度数据确认机制 | P0 | 财务确认月度数据，锁定报表 |
| F002 | 撤回确认 | P1 | 已确认月份恢复为未确认 |
| F003 | 未确认数据自动变更 | P0 | 未确认月份油费变更自动更新 |
| F004 | 变更影响检测 | P0 | 自动检测油费变更是否影响已确认月份 |
| F005 | 变更提醒与确认更新 | P0 | 已确认数据变更的提醒和确认处理 |
| F006 | 周期变更影响提示 | P1 | 变更影响满到满周期时的明确提示 |
| F007 | 变更记录查询 | P2 | 查询变更提醒和处理历史 |

### 3.2 功能详细说明

#### F001：月度数据确认机制

##### 数据状态

| 状态 | 说明 | 数据变更行为 |
|------|------|-------------|
| 未确认 | 报表计算完成，财务尚未确认 | 油费变更自动更新报表 |
| 已确认 | 财务已核算并确认 | 油费变更不自动更新，生成变更提醒 |

##### 确认规则

- 确认粒度：按**月份整体**确认（该月所有车辆一起确认）
- 确认权限：财务人员/车队管理员
- 确认前提：该月数据已生成（有满到满计算结果）
- 确认操作记录操作人和操作时间

##### 自动变更规则

| 月份状态 | 油费变更行为 |
|----------|-------------|
| 当前月（未结算月） | 始终为未确认，自动更新 |
| 上月（结算中，未确认） | 自动更新（适应司机上报节奏） |
| 已确认月份 | 不自动更新，生成变更提醒 |

##### 异常处理

| 场景 | 处理方式 |
|------|----------|
| 确认时该月无数据 | 不允许确认，提示"该月无油耗数据" |
| 确认时存在待处理的变更提醒 | 提示先处理待处理变更再确认 |

---

#### F002：撤回确认

##### 业务规则

- 已确认月份支持"撤回确认"，恢复为"未确认"状态
- 撤回后该月份的待处理变更提醒**自动执行更新**
- 撤回操作记录操作人和操作时间

---

#### F003：未确认数据自动变更

##### 业务规则

- 司机新增/修改油费记录时，如果影响的月份状态为"未确认"，系统按正常流程自动更新报表数据
- 自动更新包括：
  - 实际升数重算
  - 平均油价重算
  - 升数差额重算
  - 奖惩金额重算
- 如果油费为加满记录，且影响满到满周期分段，自动更新包含周期重新划分

---

#### F004：变更影响检测

##### 检测流程

```
司机新增/修改油费记录（已通过油费上报审批）
    ↓
系统判断该油费记录所属满到满周期归属的月份
    ↓
检查归属月份的确认状态
    ↓
┌─ 月份状态 = 未确认 → 自动更新，流程结束
│
└─ 月份状态 = 已确认 → 继续检测影响范围
        ↓
    检测是否为加满记录
        ↓
    ┌─ 非加满记录 → 仅影响当前月份实际升数
    │
    └─ 加满记录 → 检测是否影响满到满周期分段
            ↓
        ┌─ 不影响周期 → 仅影响当前月份实际升数
        │
        └─ 影响周期 → 检测相邻月份是否也受影响
                ↓
            生成变更提醒（含完整影响范围）
```

##### 检测项

| 检测项 | 说明 |
|--------|------|
| 加油时间归属 | 油费记录所属的满到满周期归属哪个月份 |
| 是否加满 | 是否为 `tank_full = true` 的记录 |
| 归属月份状态 | 该月份是否已确认 |
| 周期影响 | 加满记录是否影响周期节点的选取 |
| 跨月影响 | 周期变更是否连带影响相邻月份 |

---

#### F005：变更提醒与确认更新

##### 处理流程

```
变更影响检测 → 影响已确认月份
    ↓
生成变更提醒
    ↓
通知管理员/财务（系统消息/待办事项）
    ↓
管理员查看变更详情和影响范围
    ↓
┌─ 确认更新
│     ↓
│   系统自动重算相关月份数据
│     ↓
│   更新报表，提醒标记为"已处理"
│
└─ 取消（不处理）
      ↓
    报表数据暂不更新
      ↓
    提醒保留，待后续处理
```

##### 变更详情展示

| 信息项 | 内容 |
|--------|------|
| 变更概要 | 司机姓名 + 变更类型（新增/修改） |
| 变更详情 | 油费记录：时间、加油站、金额、升数、是否加满 |
| 影响月份 | 受影响的已确认月份列表 |
| 周期影响 | 是否影响满到满周期，影响哪些月份的周期 |
| 数据对比 | 变更前后的对比表（见下表） |

**数据对比表格式**：

| 对比项 | 变更前 | 变更后 | 差异 |
|--------|--------|--------|------|
| 实际升数 | X升 | Y升 | +/-Z升 |
| 标准升数 | A升 | A升 | 不变 |
| 升数差额 | M升 | N升 | +/-P升 |
| 奖惩金额 | ¥Q | ¥R | +/-¥S |

##### 满到满周期影响的特殊提示

当变更影响满到满周期分段时，详情页需增加醒目提示：

> ⚠️ 该变更会影响满到满周期分段，确认更新后将对 **X月** 和 **Y月** 的满到满数据都产生影响。

---

#### F006：周期变更影响提示

##### 业务规则

当油费变更为加满记录且影响满到满周期时：

- 展示当前周期：开始时间 → 结束时间
- 展示变更后周期：新的开始时间 → 新的结束时间
- 展示受影响月份的数据预估变化
- 如果涉及两个月份，两个月的影响都要展示

---

#### F007：变更记录查询

##### 查询条件

| 筛选项 | 说明 |
|--------|------|
| 月份 | 按影响月份筛选 |
| 车辆 | 按车牌号筛选 |
| 司机 | 按司机姓名筛选 |
| 处理状态 | 待处理/已确认更新/未处理 |

##### 列表展示字段

| 字段 | 说明 |
|------|------|
| 变更时间 | 司机提交油费变更的时间 |
| 变更人 | 司机姓名 |
| 变更内容 | 油费记录摘要 |
| 影响月份 | 受影响的月份 |
| 处理状态 | 待处理/已确认更新/未处理 |
| 处理人 | 操作人 |
| 处理时间 | 操作时间 |

### 3.3 非功能需求

- **实时性**：油费变更后，影响检测和通知需实时触发
- **数据一致性**：确认更新后的重算逻辑与自动计算逻辑一致
- **可追溯性**：所有确认、撤回、变更处理操作均有记录

## 4. 交互设计

### 4.1 页面流程图

```
月报表列表页
  ├── 未确认月份
  │     ├── 查看数据（正常查看）
  │     ├── 确认按钮 → 二次确认弹窗 → 确认
  │     └── 数据自动更新（司机油费变更时）
  │
  ├── 已确认月份
  │     ├── 查看数据（带"已确认"标识）
  │     ├── 撤回确认按钮 → 二次确认弹窗 → 撤回
  │     └── 数据锁定（变更走提醒确认流程）
  │
  └── 变更提醒入口（待办事项/消息中心）
        └── 变更详情页
              ├── 变更内容 + 影响范围 + 数据对比
              ├── 周期影响提示（如有）
              ├── 确认更新 → 重算数据
              └── 取消 → 暂不处理，提醒保留
```

### 4.2 界面说明

#### 月报表列表页（增强）

| 元素 | 说明 |
|------|------|
| 确认状态列 | 展示"未确认"/"已确认"状态标签 |
| 确认按钮 | 未确认状态时可用 |
| 撤回确认按钮 | 已确认状态时可用 |
| 待处理数量角标 | 在消息入口展示待处理变更数量 |

#### 确认弹窗

| 元素 | 说明 |
|------|------|
| 标题 | "确认[YYYY年MM月]油耗数据" |
| 提示文案 | "确认后该月数据将被锁定，司机后续油费变更需人工确认后才能更新报表。是否确认？" |
| 取消按钮 | 关闭弹窗 |
| 确认按钮 | 执行确认操作 |

#### 变更详情页

| 区域 | 内容 |
|------|------|
| 变更概要 | 司机姓名、变更类型、变更时间 |
| 变更详情 | 油费记录：加油时间、加油站、金额、升数、是否加满 |
| 影响范围 | 受影响月份、是否影响周期分段 |
| 数据对比 | 变更前后的关键指标对比表 |
| 周期影响提示 | 醒目提示影响哪些月份的满到满数据（如有） |
| 操作按钮 | 确认更新 / 取消 |

### 4.3 交互规则

- 确认/撤回操作均需二次确认
- 变更提醒列表按时间倒序排列，待处理优先展示
- 取消不处理的提醒保留在列表中，可随时再次查看处理
- 同一月份存在多个待处理变更时，支持逐条处理

### 5.2 数据流转

```
司机新增/修改油费（已通过油费上报审批）
    ↓
变更影响检测
    ↓
┌─ 影响未确认月份
│     ↓
│   自动更新报表数据
│
└─ 影响已确认月份
      ↓
    生成变更提醒（oil_change_notification）
      ↓

      ↓
    ┌─ 确认更新 → 更新报表数据 → 标记已处理

```

## 7. 验收标准

### 7.1 功能验收

#### 数据确认机制
- [ ] 财务人员可对某月数据执行"确认"操作
- [ ] 确认后月份状态变为"已确认"，界面有明显标识
- [ ] 已确认月份支持撤回确认
- [ ] 撤回后待处理变更自动执行更新

#### 自动变更规则
- [ ] 当前月和未确认的上月：油费变更自动更新报表
- [ ] 已确认月份：油费变更不自动更新，生成变更提醒

#### 变更提醒与确认更新
- [ ] 影响已确认月份时自动生成变更提醒并通知管理员
- [ ] 提醒信息包含：变更内容、影响范围、数据对比
- [ ] 影响满到满周期时，明确提示"确认更新后会对X月和Y月的满到满数据都产生影响"
- [ ] 确认更新后自动重算相关月份数据
- [ ] 取消不处理时报表数据不变，提醒保留

#### 变更记录
- [ ] 可按月份、车辆、司机查询变更记录
- [ ] 记录包含变更详情和处理结果

### 7.2 性能验收

- [ ] 变更影响检测响应时间 < 2秒
- [ ] 确认更新后数据重算不影响系统正常使用
