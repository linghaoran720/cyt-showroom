## 1. 概述

### 1.1 项目背景

货运经营者在日常运单录入中面临两个核心痛点：

**痛点一：数据源不一致**
客户通过微信图片、文字消息、邮件Excel等多种渠道发送订单，格式各异，字段命名不统一，调度员需要人工理解和转译后才能录入系统。

**痛点二：录入时间成本高**
每张运单需要手动逐字段录入系统，单张耗时3-10分钟，批量订单时效率极低，且容易出错。

### 1.2 项目目标

| 目标 | 指标 |
|------|------|
| 录入效率提升 | 单张运单录入时间从3-10分钟降至<1分钟 |
| 识别准确率 | AI识别准确率≥95% |
| 批量处理效率 | 10张运单处理时间从30-100分钟降至<10分钟 |
| 用户体验 | 调度员无需学习新格式，任意输入均可识别 |

**价值主张**：让调度员从"数据搬运工"变成"数据确认者"——无论客户用什么格式发来订单，系统自动识别、自动映射、自动填单，调度员只需确认和补充关键信息，一键创建运单。

### 1.3 项目范围

**包含（In Scope）**
- 图片+文字识别（单个/批量运单）
- 纯文字识别（单个/批量运单）
- Excel批量识别
- 字段智能映射（含冲突处理）
- 映射规则管理（查看、搜索、删除）
- 单/多运单自动判断

**不包含（Out of Scope）**
- 语音识别
- 费用单识别（Phase 3）
- 合同/发票识别（Phase 3）
- 映射规则批量编辑/导入导出（Phase 2）
- 自定义Prompt配置（Phase 3）

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| 识别字段 | AI从输入内容中提取的字段，如"客户"、"发货地" |
| 系统字段 | 财运通运单创建配置中定义的标准字段 |
| 字段映射 | 识别字段与系统字段的对应关系 |
| 映射规则 | 已保存的识别字段→系统字段对应关系，全集团共享 |
| 置信度 | AI对识别结果准确性的评估分值（0-1） |
| 双表头表格 | 批量识别结果展示方式：一级表头=系统字段，二级表头=识别字段 |
| 两阶段识别 | 阶段1：判断单/多运单；阶段2：提取字段数据 |

## 2. 用户分析

### 2.1 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 调度员 | 日常接收客户订单并录入系统 | 快速录单，减少手动输入 |
| 系统管理员 | 维护系统配置和规则 | 管理映射规则，保证识别准确性 |

权限控制：功能入口根据系统配置的权限决定，不限制特定角色。

### 2.2 用户价值

| 价值维度 | 当前状态 | 使用后 |
|----------|----------|--------|
| 录入效率 | 每张运单手动录入3-10分钟 | AI识别+确认，<1分钟完成 |
| 数据准确性 | 人工转录易出错，错误率约5-10% | AI识别+人工校验，准确率≥95% |
| 多格式适配 | 需要人工理解不同格式，重新录入 | 图片/文字/Excel统一识别，无需转译 |
| 字段映射 | 每次手动对应客户字段与系统字段 | 映射规则自动积累，越用越准 |
| 批量处理 | 10张运单需30-100分钟 | 批量上传，统一确认，效率提升80%+ |

### 2.3 使用场景

| 场景 | 触发条件 | 用户操作 |
|------|----------|----------|
| 图片订单 | 客户微信发来运单照片 | 上传图片→AI识别→确认→创建运单 |
| 文字订单 | 客户微信发来文字订单信息 | 复制粘贴→AI识别→确认→创建运单 |
| Excel订单 | 客户邮件发来Excel订单表 | 上传Excel→AI识别→批量确认→批量创建 |

### 3.2 功能详细说明

#### F001：智能识别入口

**功能描述**
在创建运单页面提供"智能识别"按钮，点击后弹出识别弹窗，支持三种输入模式：文字建单（默认）、图片建单、Excel建单。

**业务规则**
1. 默认模式为文字建单
2. 同一会话内保持用户选择的模式
3. 三种模式通过底部Tab切换

---

#### F002：图片识别-单个运单

**功能描述**
用户上传1-2张运单图片（可选填补充信息），AI识别后以左右分屏方式展示原始图片和识别结果。

**业务规则**
1. 支持JPG/PNG/PDF格式，单张≤10MB
2. 最多上传2张图片
3. 支持拖拽上传
4. 补充信息最多1000字符（可选）
5. 识别采用两阶段流程：阶段1判断单/多运单，阶段2提取字段数据
6. 不管置信度多少，均进入阶段2（极致降低用户门槛）
7. 识别完成后展示左右分屏界面

**左右分屏说明**

| 区域 | 内容 | 交互 |
|------|------|------|
| 左侧 | 原始图片列表、补充信息 | 只读，图片可点击放大 |
| 右侧 | 识别字段、映射下拉、识别内容 | 可编辑内容，可调整映射 |

**右侧识别结果字段分组**
- 自动映射字段：置信度≥30%，已自动映射到系统字段
- 补充信息带入：从补充信息中识别的字段
- 需手动确认：置信度<30%，映射为空，需用户手动选择

**异常处理**
- 识别超时（>10秒）：提示"识别超时，请重试"
- 图片格式不支持：提示支持的格式
- 图片过大：提示文件大小限制

---

#### F003：文字识别-单个运单

**功能描述**
用户粘贴任意格式的运单文字，AI识别后以左右分屏方式展示原始文字和识别结果。

**业务规则**
1. 支持任意格式文字，最多1000字符
2. 识别流程同图片识别（两阶段）
3. 左侧展示原始文字（只读），右侧展示识别结果

---

#### F004：智能字段映射

**功能描述**
AI识别字段后，根据映射规则优先级自动映射到系统字段，支持用户手动调整。

**映射优先级**
```
1. 有映射规则 → 直接应用（不考虑置信度）
2. 无映射规则 + 置信度≥30% → 自动映射
3. 无映射规则 + 置信度<30% → 留空，需用户手动选择
```

**业务规则**
1. 映射下拉框显示所有系统字段，不显示置信度
2. 按Delete键可清空映射（识别内容保留，提交时该字段不带入运单）
3. 互斥映射：选择新字段后，原映射自动清空
4. 批量识别时，映射调整默认自动应用到所有运单

---

#### F005：映射冲突处理

**功能描述**
当用户调整的映射关系与已保存的全局映射规则冲突时，弹窗提示用户选择处理方式。

**冲突触发条件**
用户将识别字段A的映射从"系统字段X"改为"系统字段Y"，且已存在"A→X"的全局映射规则。

**弹窗内容**
- 显示原映射：识别字段 → 原系统字段
- 显示新映射：识别字段 → 新系统字段
- 选项一：仅本次生效（不更新映射规则）
- 选项二：更新映射规则（全集团生效）

**业务规则**
1. 无冲突时，映射调整静默应用，不弹窗
2. 有冲突时必须用户确认后才能继续
3. 选择"更新映射规则"后，全集团所有用户的后续识别均使用新规则

---

#### F006：图片识别-批量运单

**功能描述**
用户上传多组运单图片（每组最多2张），AI识别后以双表头表格展示所有运单的识别结果。

**业务规则**
1. 最多添加10个运单上传区
2. 每个运单最多2张图片
3. 磅单场景：通过时间戳判断装货/卸货（时间早的为装货）
4. 普通运单：通过关键字段相似度>80%判断是否同一运单
5. 识别结果以双表头表格展示（详见F009）

---

#### F007：文字识别-批量运单

**功能描述**
用户粘贴包含多个运单的文字，AI自动判断运单数量，识别后以双表头表格展示。

**业务规则**
1. 支持任意格式文字，最多1000字符
2. AI自动判断文字中包含的运单数量
3. 识别结果以双表头表格展示（详见F009）

---

#### F008：Excel批量识别

**功能描述**
用户上传Excel文件，AI解析表头并映射到系统字段，以双表头表格展示识别结果。

**业务规则**
1. 支持.xlsx和.xls格式
2. 必须有表头（第一行为字段名）
3. 一行 = 一个运单
4. 最多100行，超过自动截取前100行并提示
5. 一级表头=系统字段，二级表头=Excel表头字段
6. 自动映射到置信度最高的系统字段

---

#### F009：批量识别表格交互

**功能描述**
所有批量识别（图片/文字/Excel）统一使用双表头表格模式，在当前页面弹窗展示。

**弹窗布局**
- 顶部：原始内容展示区（只读）——图片缩略图 / 文字内容 / Excel文件名
- 中部：双表头表格区域
- 底部：操作按钮（导出Excel、批量创建运单）

**双表头说明**

| 层级 | 内容 | 交互 |
|------|------|------|
| 一级表头 | 系统字段（来自运单创建配置） | 只读，不可修改 |
| 二级表头 | 识别字段（AI识别结果） | 可点击[×]清空，可下拉选择其他识别字段 |
| 数据行 | 各运单的识别内容 | 可任意编辑 |

**二级表头互斥规则**
选择新识别字段后，该字段在其他列的映射自动清空。

**数据行操作**
- 所有单元格可直接编辑
- 车牌号字段：输入触发模糊搜索
- 地址字段：调用高德地图API联想
- 每行末尾提供删除按钮，删除后行数实时更新

**批量创建结果**
- 弹窗展示每条运单的创建结果（成功/失败）
- 失败时显示失败原因
- 支持导出Excel（含创建结果）

---

#### F010：映射规则管理

**功能描述**
系统管理员可查看、搜索、删除全集团的映射规则。

**入口**
- **主路径**：系统设置 → 智能识别 → 映射规则管理
- **补充入口（智能录单）**：创建运单 · **智能录单助手（财财）** PC 底部工具条 **「识别配置」** → **Tab「映射配置」**（与 **Tab「简称配置」** 同属一弹窗），打开与本页 **同一套规则数据** 的管理能力（全屏页或大型弹窗/抽屉由实现定稿，数据与权限与主路径一致）。详见 **[2604-智能录单-运单文本识别-PRD-v1.0.md](./智能识别-录单/2604-智能录单-运单文本识别-PRD-v1.0.md) §5.4**。

**页面功能**
- 列表展示：识别字段、系统字段、使用次数（含可视化进度条）、最后使用时间、创建人
- 搜索：支持按识别字段或系统字段名称搜索
- 类型筛选：全部 / 运单 / 费用
- 删除：点击删除按钮，弹窗确认后删除

**删除确认弹窗**
- 显示：识别字段、系统字段、使用次数
- 警告：删除后下次识别到该字段需重新映射

### 3.3 非功能需求

**性能要求**
- 单个运单识别响应时间≤5秒
- 批量识别（10个运单）响应时间≤30秒
- 页面加载时间<2秒

**准确性要求**
- AI识别准确率≥95%
- 字段映射准确率≥90%（有历史规则时）

**安全要求**
- 映射规则按集团（tenant）隔离，不同集团数据不互通
- 上传图片/文件在识别完成后不长期存储
- 操作日志记录（映射规则变更）

**兼容性要求**
- 浏览器：Chrome 90+、Safari 14+、Edge 90+
- 分辨率：1280×720及以上

## 4. 交互设计

```
创建运单页面

识别弹窗（文字/图片/Excel 三种模式）
    ↓ 点击"开始识别"
    ↓
识别中（Loading）
    ↓

└─────────────────────────────────┘
┌─────────────────────────────────┐
│  批量识别结果（双表头表格弹窗）   │  → 编辑确认 → 批量创建运单
└─────────────────────────────────┘
```

- 点击后弹出识别弹窗，底部Tab切换三种模式

**页面2：单个运单识别结果**

- 左侧：原始图片/文字（只读）
- 右侧：识别字段表格（映射下拉 + 内容编辑）

**页面3：批量识别结果**

**页面4：映射规则管理**

- 表单必填项：标红星提示
- 映射为空的列：橙色边框高亮提示（需手动确认）

## 5. 数据需求

### 5.1 数据模型

**映射规则表（mapping_rules）**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| rule_id | varchar(36) | 是 | 主键，UUID |
| tenant_id | varchar(36) | 是 | 集团ID，多租户隔离 |
| source_field | varchar(100) | 是 | 识别字段名称 |
| target_field | varchar(100) | 是 | 系统字段ID |
| target_field_name | varchar(100) | 是 | 系统字段显示名称 |
| usage_count | int | 是 | 使用次数，默认0 |
| created_by | varchar(36) | 是 | 创建人user_id |
| created_at | datetime | 是 | 创建时间 |
| last_used | datetime | 否 | 最后使用时间 |

**映射变更日志表（mapping_change_logs）**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| log_id | varchar(36) | 是 | 主键，UUID |
| tenant_id | varchar(36) | 是 | 集团ID |
| source_field | varchar(100) | 是 | 识别字段 |
| old_target | varchar(100) | 否 | 原系统字段 |
| new_target | varchar(100) | 是 | 新系统字段 |
| change_type | varchar(20) | 是 | once（仅本次）/ global（全局更新）|
| operator | varchar(36) | 是 | 操作人user_id |
| created_at | datetime | 是 | 变更时间 |

### 5.2 数据流转

```
用户上传内容
    ↓
AI识别 → 返回识别字段列表（含置信度）
    ↓
查询映射规则表 → 匹配已有规则
    ↓
展示识别结果（已映射 / 待确认）
    ↓
用户确认/调整映射
    ↓
检测冲突 → 弹窗确认（如有冲突）
    ↓
创建运单 → 更新映射规则（如选择全局更新）
```

## 7. 验收标准

### 7.1 P0 功能验收

- [ ] F001：点击"智能识别"按钮，弹窗正常打开，三种模式Tab切换正常
- [ ] F002：上传图片后点击识别，5秒内返回左右分屏结果；识别字段可编辑；映射下拉可选择；按Delete键可清空映射
- [ ] F003：粘贴文字后点击识别，5秒内返回左右分屏结果；交互同F002
- [ ] F004：置信度≥30%的字段自动映射；置信度<30%的字段留空；有历史规则的字段直接应用规则
- [ ] F005：调整映射与已有规则冲突时弹窗提示；选择"仅本次生效"不更新规则；选择"更新映射规则"后全集团生效

### 7.2 P1 功能验收

- [ ] F006：批量上传多组图片，识别后双表头表格正常展示；可删除单行运单
- [ ] F007：粘贴多运单文字，AI自动判断运单数量，双表头表格正常展示
- [ ] F008：上传Excel，表头自动映射，双表头表格正常展示；超100行时截取并提示
- [ ] F009：二级表头可删除/下拉选择；互斥映射正常；数据行可编辑；车牌号模糊搜索正常；地址联想正常；批量创建结果弹窗展示成功/失败

### 7.3 P2 功能验收

- [ ] F010：映射规则列表正常展示；搜索功能正常；删除确认弹窗正常；删除后列表实时更新

### 7.4 性能验收

- [ ] 单个运单识别响应时间≤5秒（正常网络环境）
- [ ] 批量10个运单识别响应时间≤30秒
- [ ] 页面加载时间<2秒

### 7.5 异常场景验收

- [ ] 上传不支持格式的文件，提示错误信息
- [ ] 识别超时，提示重试
- [ ] Excel超100行，截取前100行并提示
- [ ] 网络异常，提示网络错误

## 8. 数据埋点

### 8.1 关键事件

| 事件名称 | 触发时机 | 关键参数 |
|----------|---------|----------|
| recognition_start | 点击"开始识别" | input_type（image/text/excel）, waybill_count |
| recognition_success | 识别成功返回结果 | input_type, duration_ms, field_count |
| recognition_fail | 识别失败 | input_type, error_type |
| mapping_adjust | 用户手动调整映射 | source_field, old_target, new_target |
| mapping_conflict_shown | 映射冲突弹窗展示 | source_field |
| mapping_conflict_choice | 用户选择冲突处理方式 | choice（once/global） |
| waybill_create_from_recognition | 从识别结果创建运单 | input_type, waybill_count, success_count |

### 8.2 关键指标

- 识别功能使用率（使用识别建单 / 总建单数）
- 识别成功率（识别成功 / 识别发起）
- 映射调整率（有手动调整的识别 / 总识别次数）
- 映射冲突率（触发冲突弹窗 / 总映射调整次数）
- 批量创建成功率（成功创建运单数 / 总识别运单数）
