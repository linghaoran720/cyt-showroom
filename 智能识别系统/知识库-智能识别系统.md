# 财运通智能识别系统 - 知识库文档

## 文档概述

本知识库整理了财运通SaaS系统中基于AI大模型实现的智能识别能力，主要覆盖司机运输过程中产生的费用凭证和货物交接凭证的自动采集识别功能。

**最后更新时间**: 2026-04-13

---

## 一、系统架构概览

### 1.1 核心能力

财运通智能识别系统包含以下三大核心识别能力：

1. **在途费用识别** - 自动识别司机上报的费用凭证（油费、路桥费、尿素费、电费等）
2. **磅单识别** - 自动识别装卸货过程中的磅单信息
3. **重复报销检测** - 基于支付凭证的智能查重

### 1.2 技术架构

- **大模型供应商**: 豆包（doubao-1-5-vision-pro-32k）、通义千问（qwen-vl-max）
- **识别方式**: 同步识别（司机端点击）+ 异步识别（后台自动）
- **成本控制**:
  - Input: 0.003元/1000tokens
  - Output: 0.009元/1000tokens
  - 单张图片约0.00567元
  - 月均1000单约28.35元

## 二、在途费用识别

### 2.1 功能版本演进

#### 2505版本 - 重复报销识别
**需求编号**: CYTRD-19826
**上线时间**: 2025年5月

**核心功能**:
- 识别微信/支付宝支付截图中的金额、时间、支付编号
- 自动标记"疑似重复报销"
- 支持多张账单截图的合并识别

**识别内容**:
- 支付金额（优先级：小票 > 油枪 > 付款截图）
- 支付时间（精确到秒）
- 支付编号（订单号、交易单号、转账单号）

**异常标签**:
1. 疑似重复报销 - 全集团近60天内识别的【支付时间+金额】完全相同
2. 无付款截图 - 未识别到微信/支付宝账单截图
3. 报销金额不符 - 识别金额 < 司机填写金额的99%

#### 2507版本 - 油费识别
**需求编号**: CYTRD-21316
**上线时间**: 2025年7月

**核心功能**:
- 识别加油小票、油枪照片、支付截图、车辆码表
- 自动填充：金额、时间、升数、单价、里程
- 支持多笔加油记录合并

**识别优先级**:
```
金额: 付款截图 → 小票 → 油枪
时间: 付款截图 → 小票 → 油枪水印
升数: 小票 → 油枪
码表: 唯一数据源
单价: 金额/升数（自动计算）
```

**特殊场景处理**:
- 多笔加油：识别多张小票或油枪照片，自动求和
- 加气场景：支持LNG/CNG加气记录识别（单位：千克）
- 撬装加油：识别油桩电子屏信息

**客户调研数据**（2025年3月）:
- 司机上报量：526,468单（4.29亿元）
- 油费占比：178,191单（3.61亿元，占84%）
- 驳回率：约10%单量，21%金额

#### 2511版本 - 电费识别
**需求编号**: CYTRD-22658
**上线时间**: 2025年11月

**核心功能**:
- 识别充电付款账单、充电订单详情、充电系统结算界面
- 自动填充：金额、度数、电费、服务费、充电时间、SOC等

**识别票据类型**:
- A类：充电付款及退款账单（微信/支付宝）
- B类：充电订单详情页面
- C类：充电系统结算界面
- D类：车辆码表

**识别优先级**:
```
金额: A → B → C
发生时间: A → B → C
充电量: B → C
电费/服务费: B → C
充电前后SOC: B → C
```

### 2.2 交互流程

#### 司机端上报流程
1. 选择费用类型 → 选择费用名称
2. 上传付款照片（最多6-8张）
3. 点击"不想填？点我智能识别"按钮
4. Loading识别中（最长10秒）
5. 识别成功/失败提示
6. 核对并修改识别结果
7. 提交费用

**识别次数限制**: 每个单据最多调用2次

#### 员工端审核流程
1. 查看费用明细列表
2. 查看AI识别结果和异常标签
3. 点击异常标签查看详情
4. 批量筛选异常单据
5. 审核通过/驳回

### 2.3 数据字段

**费用明细列表新增字段**:
- `AI识别`: 显示"金额；升数/度数；时间；里程"
- `异常报销提醒`: 显示标签（疑似重复、无付款截图、报销内容不符）

**识别结果存储**:
- 识别支付金额（多个取合计）
- 识别支付时间（多个用逗号分隔，精确到秒）
- 识别升数/度数（多个取合计）
- 当前里程（不唯一不取）
- 油价/电价（自动计算）

## 三、磅单识别

### 3.1 功能版本演进

#### 20250722版本 - 磅单识别辅助司机填写
**需求编号**: CYTRD-23214
**上线时间**: 2025年7月

**核心功能**:
- 司机端点击"帮我识别"自动填充磅单信息
- 员工端审核页面显示识别结果
- 支持装货/卸货磅单识别

**识别字段**:

装货磅单:
- 装货过磅时间
- 装车皮重/毛重/净重
- 装车单号
- 货物名称

卸货磅单:
- 卸货过磅时间
- 卸车皮重/毛重/净重
- 卸车单号
- 货物名称

**新增运单字段**:
- 装车单号（支持全局搜索、导出、打印）
- 卸车单号（支持全局搜索、导出、打印）

#### 20251030版本 - 磅单识别优化
**需求编号**: Q4-11月
**上线时间**: 2025年10月

**核心优化**:
1. **自动填充**: 司机提交后，后台自动识别并填充空字段
2. **AI对比提醒**:
   - 紫色标签"与AI一致" - 司机填写与AI识别一致
   - 红色提示icon - 司机填写与AI识别不一致，可点击替换
3. **图片旋转优化**: 解决旋转后识别准确率下降问题

**交互优化**:
- 识别后自动填充到对应字段
- 显示"AI填写"标记
- 支持一键替换为AI识别结果

#### 2601版本 - 磅单识别可视化Prompt配置
**需求编号**: CYTRD-23614
**上线时间**: 2026年1月

**核心功能**:
- 客户自助配置个性化Prompt
- 测试工具验证识别效果
- 自定义字段映射关系

**配置流程**:
1. 上传测试磅单图片
2. 查看识别结果（6个字段）
3. 如不符合要求，标记字段对应关系
4. 系统自动生成个性化Prompt
5. 保存配置（最多5个自定义磅单）

**字段映射配置**:
- 装货过磅时间 → 装货过磅时间 / 发车时间
- 装车单号 → 装车单号 / 委托单号
- 卸货过磅时间 → 卸货过磅时间 / 到达时间
- 卸车单号 → 卸车单号 / 到达备注

### 3.2 识别Prompt示例

**装车回单识别Prompt**:
```
你是一个物流方面的专家，你的任务是：
根据用户输入的图片，提取图片中的以下字段的值。并按"字段名称：字段值"的格式输出
- 装货过磅时间
- 装车皮重
- 装车毛重
- 装车净重
- 装车单号
- 货物名称

输出时注意以下几点：
- 请用最快的速度输出
- 不要输出思考过程，直接输出识别出来的结果
- 输出各字段的结果不要重复
```

**识别效率要求**: 建议5秒以下

### 3.3 常见识别问题

| 异常Case | 解决方案 |
|---------|---------|
| 装卸车磅单一起提交 | 优化Prompt明确区分装卸货 |
| 货物名称不同 | 优先识别最明显的货物名称 |
| 漏掉磅单号 | 优化位置引导，特别注意表格外数字 |
| 手写体识别错误 | 不确定的不识别，避免误填 |
| 图片旋转后准确率下降 | 研发优化图片预处理 |
| 多个号码混淆 | 优化Prompt明确磅单号特征 |

## 八、技术实现要点

### 8.1 识别流程

**同步识别**（司机端点击）:
1. 司机上传图片
2. 点击"智能识别"按钮
3. 调用大模型API（最长10秒）
4. 返回识别结果并填充字段
5. 司机核对并提交

**异步识别**（后台自动）:
1. 司机提交费用
2. 后台自动调用大模型
3. 识别结果用于审核和查重
4. 不更新司机已提交的数据

### 8.2 查重逻辑

**重复判定条件**:
- 时间范围: 近60天（根据上报时间）
- 匹配字段: 识别支付时间 + 识别金额
- 审核状态: 待审核、审核中、已通过
- 范围: 全集团

**多张账单去重**:
- 时间间隔 < 35秒 且 金额相同 → 判定为重复
- 仅保留一条记录的金额
- 所有时间均记录

### 8.3 异常处理

**识别失败场景**:
- 识别超时（>10秒）: 提示"识别超时，请手动填写"
- 未识别到内容: 提示"未识别到信息，请手动填写"
- 识别次数超限: 提示"单笔费用最多调用2次"

**图片质量要求**:
- 图片清晰度: 模糊或倾斜影响准确率
- 图片大小: 超大文件可能导致识别报错
- 图片旋转: 需要预处理优化

### 8.4 智能录单 · 字段映射（F004 / F005 / F010 索引）

> **完整规则与验收**以 **[2604-智能识别平台-PRD-v1.0.md](./2604-智能识别平台-PRD-v1.0.md) §3.2** 为准。下列条目便于本知识库内检索，与 **创建运单 · 智能录单助手（财财）**、**运单文本识别** PRD 交叉引用。

#### F004：智能字段映射

- **含义**：模型/规则输出的 **识别字段名**（含客户口语、票据上的多种叫法）按优先级 **自动映射** 到 **财运通系统字段**；用户可在识别结果侧 **手动调整** 映射目标。
- **优先级（摘要）**：已有 **映射规则** → 直接应用；无规则且置信度≥30% → 自动映射；无规则且置信度&lt;30% → 留空待用户选择。
- **自动规则质量（与 F010 一致）**：自动推断/写入前应结合 **列标题（或字段名）与单元格内容** 判断；**「12345测试」** 等与业务无语义的映射 **不自动入库**，除非用户手动确认或手动添加规则。
- **交互要点**：映射下拉展示系统字段；互斥映射、Delete 清空等见平台 PRD。

#### F005：映射冲突处理

- **含义**：用户将某识别字段的映射从既有全局规则 **改为** 另一系统字段时，若与已存规则冲突，须 **弹窗确认**。
- **选项**：**仅本次生效**（不写回规则库）；**更新映射规则**（全集团后续识别沿用新规则）。

#### F010：映射规则管理

- **含义**：管理员查看、**搜索**、**删除** 本集团 **识别字段 → 系统字段** 规则列表；可展示 **使用次数**、**最近使用时间** 等（见平台 PRD 列表列定义）。
- **主入口**：**系统设置 → 智能识别 → 映射规则管理**；**补充入口**：智能录单助手 PC 底栏 **「识别配置」→「映射配置」Tab**（与 **[2604-智能录单-运单文本识别-PRD-v1.0.md](./智能识别-录单/2604-智能录单-运单文本识别-PRD-v1.0.md) §5.4** 一致）。
- **自动规则质量**：自动写入映射表前应结合 **列标题/字段名与内容** 判断；**「12345测试」** 等与业务无语义的映射 **不自动入库**，除非用户手动添加（同智能录单 PRD §5.4.3）。
