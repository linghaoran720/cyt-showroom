## 一、系统定位与价值

### 1.1 系统定位

财运通智能识别系统是基于大模型（豆包/通义千问）的AI能力，为财运通TMS系统提供的智能化数据采集和识别服务。主要解决司机在运输过程中手动录入数据的痛点，提升数据准确性和审核效率。

### 1.2 核心价值主张

**对司机**:
- 上报时间从50秒降至22秒，节省56%时间
- 无需手动填写费用类型、时间、金额等字段
- 降低使用门槛，提升上报意愿

**对审核人员**:
- 审核效率提升30%
- 自动标记异常单据（重复、金额不符、缺凭证）
- 快速筛选和批量处理

**对企业**:
- 人力成本降低38%（典型案例：30车/人 → 50车/人）
- 重复报销损失降低（3‰ → 0.5‰）
- 数据完整性和准确性提升

## 三、功能模块详解

### 3.1 在途费用识别

#### 3.1.1 支持的费用类型

| 费用类型 | 版本 | 识别内容 | 特殊说明 |
|---------|------|---------|---------|
| 油费 | 2507 | 金额、时间、升数、单价、里程 | 支持加油/加气，多笔合并 |
| 电费 | 2511 | 金额、时间、度数、电费、服务费、SOC | 支持充电订单详情 |
| 路桥费 | 2505 | 金额、时间 | 仅支付凭证识别 |
| 尿素费 | 2505 | 金额、时间 | 仅支付凭证识别 |
| 其他费 | 2505 | 金额、时间 | 仅支付凭证识别 |

#### 3.1.2 识别流程

**同步识别（司机端）**:
```
1. 司机上传图片（最多6-8张）
2. 点击"不想填？点我智能识别"
3. 调用大模型API（最长10秒）
   ├─ 成功：显示"识别成功，请核对识别结果"
   └─ 失败：显示"识别失败，点击再试一次"
4. 自动填充识别结果到表单
5. 司机核对并修改
6. 提交费用
```

**异步识别（后台）**:
```
1. 司机提交费用（无论是否点击识别）
2. 后台自动调用大模型
3. 识别结果存储到数据库
4. 用于审核展示和查重判断
5. 不更新司机已提交的数据
```

#### 3.1.3 识别优先级规则

**油费识别优先级**:
```
金额: 付款截图 > 小票 > 油枪
时间: 付款截图 > 小票 > 油枪水印
升数: 小票 > 油枪
里程: 码表（唯一）
单价: 金额 ÷ 升数（自动计算）
```

**电费识别优先级**:
```
A = 充电付款账单（微信/支付宝）
B = 充电订单详情页面
C = 充电系统结算界面

金额: A → B → C
时间: A → B → C
度数: B → C
电费/服务费: B → C
```

#### 3.1.4 异常标签系统

| 标签名称 | 触发条件 | 显示位置 | 交互 |
|---------|---------|---------|------|
| 疑似重复报销 | 识别的【时间+金额】完全相同，60天内 | 费用明细列表 | 点击查看重复单据 |
| 无付款截图 | 未识别到微信/支付宝账单 | 费用明细列表 | 无交互 |
| 报销内容不符 | 识别金额<填写金额99% 或 时间/升数/里程不符 | 费用明细列表 | 无交互 |

### 3.2 磅单识别

#### 3.2.1 识别字段

**装货磅单**:
- 装货过磅时间
- 装车皮重/毛重/净重
- 装车单号
- 货物名称

**卸货磅单**:
- 卸货过磅时间
- 卸车皮重/毛重/净重
- 卸车单号
- 货物名称

#### 3.2.2 识别流程

**司机端**:
```
1. 上传磅单图片
2. 点击"帮我识别"按钮
3. 识别所有已上传图片（5秒内）
4. 自动填充识别结果
5. 司机核对并提交
```

**员工端审核**:
```
1. 查看磅单识别结果（只读展示）
2. 对比司机填写与AI识别
   ├─ 一致：显示紫色"与AI一致"标签
   └─ 不一致：显示红色提示icon，可点击替换
3. 审核通过/驳回
```

#### 3.2.3 个性化Prompt配置（2601版本）

**配置流程**:
```
1. 上传测试磅单图片
2. 系统识别并显示6个字段结果
3. 如不符合要求：
   ├─ 标记异常字段的对应关系
   ├─ 标记磅单名称
   └─ 系统自动生成个性化Prompt
4. 保存配置（最多5个）
5. 后续该磅单自动使用个性化Prompt
```

**字段映射配置**:
- 装车单号 → 装车单号 / 委托单号
- 卸车单号 → 卸车单号 / 到达备注
- 装货过磅时间 → 装货过磅时间 / 发车时间
- 卸货过磅时间 → 卸货过磅时间 / 到达时间

## 五、业务规则与逻辑

### 5.1 查重逻辑

**重复判定条件**:
```javascript
// 伪代码
function isDuplicate(record1, record2) {
  // 1. 时间范围：近60天（基于上报时间）
  const daysDiff = getDaysDiff(record1.submitTime, record2.submitTime);
  if (daysDiff > 60) return false;

  // 2. 审核状态：待审核、审核中、已通过
  const validStatuses = ['待审核', '审核中', '已通过'];
  if (!validStatuses.includes(record1.status) ||
      !validStatuses.includes(record2.status)) {
    return false;
  }

  // 3. 识别时间+金额完全相同
  if (record1.recognizedTime === record2.recognizedTime &&
      record1.recognizedAmount === record2.recognizedAmount) {
    return true;
  }

  return false;
}
```

**多张账单去重**:
```javascript
// 时间间隔<35秒 且 金额相同 → 判定为重复
function deduplicateMultipleReceipts(receipts) {
  const uniqueReceipts = [];

  for (let i = 0; i < receipts.length; i++) {
    let isDup = false;
    for (let j = 0; j < uniqueReceipts.length; j++) {
      const timeDiff = Math.abs(receipts[i].time - uniqueReceipts[j].time);
      if (timeDiff < 35 && receipts[i].amount === uniqueReceipts[j].amount) {
        isDup = true;
        // 记录时间但不累加金额
        uniqueReceipts[j].times.push(receipts[i].time);
        break;
      }
    }
    if (!isDup) {
      uniqueReceipts.push({
        amount: receipts[i].amount,
        times: [receipts[i].time]
      });
    }
  }

  return uniqueReceipts;
}
```

### 5.2 审批状态变更逻辑

| 操作 | 状态变化 | 标签处理 |
|------|---------|---------|
| 审核通过 | 待审核/审核中 → 已通过 | 清除"无付款截图"和"报销内容不符"，保留"疑似重复" |
| 驳回 | 待审核/审核中 → 已驳回 | 清除所有标签（重复标签需判断关联单据） |
| 取消审核 | 已通过 → 待审核/审核中 | 保留原标签 |
| 司机修改 | 待审核 → 待审核 | 重新识别，重新打标 |

```javascript
// 每个单据最多调用2次
function canRecognize(record) {
```

## 八、已知问题与优化方向

### 8.1 已知问题

**识别准确率问题**:
1. 图片旋转后准确率下降 → 已在2510版本优化
2. 手写体识别错误 → 不确定的不识别
3. 多个号码混淆（运单号vs磅单号）→ 优化Prompt
4. 特殊字体识别错误 → 持续优化模型

**业务逻辑问题**:
1. 多节点装卸货场景复杂 → 暂不支持
2. 客户个性化需求多 → 提供Prompt配置能力
3. 部分客户不适用（员工上报为主）→ 提供黑名单机制

### 8.2 优化方向

**短期（Q1-Q2 2026）**:
1. 提升识别准确率到95%+
2. 降低识别耗时到5秒内
3. 扩展更多费用类型（维修费、停车费）
4. 优化Prompt配置界面

**中期（Q3-Q4 2026）**:
1. 支持ETC费用识别
2. 支持多节点装卸货场景
3. 提供客户自助配置能力

**长期（2027+）**:
1. 构建私有化大模型
2. 支持更多业务场景（合同识别、发票识别）

## 九、关键决策点

### 9.1 技术选型

**为什么选择豆包？**
- 成本低（0.003元/1000tokens input）
- 识别准确率高（测试准确率90%+）
- 支持图片识别
- 响应速度快（5-10秒）

**为什么不用GPT-4V？**
- 成本高（约10倍）
- 国内访问不稳定
- 合规性问题

### 9.2 产品策略

**为什么先做油费？**
- 占比最高（84%金额，33.8%单量）
- 凭证标准化程度高
- 客户痛点明确
- ROI最高

**为什么做异步识别？**
- 不影响司机提交流程
- 可以持续优化识别逻辑
- 用于审核和查重
- 降低司机端压力

**为什么限制识别次数？**
- 控制成本
- 防止滥用
- 引导司机正确上传图片

```
- 输出各字段的结果不要重复
```

### 10.3 权限配置

**在途费用识别权限**:
- 路径：在途管理 V1.0 > 在途费用识别
- 生效范围：油费、电费、路桥费、尿素费、其他费

**磅单识别权限**:
- 路径：智能能力 > 磅单识别
- 生效范围：司机端识别按钮、员工端识别结果展示
