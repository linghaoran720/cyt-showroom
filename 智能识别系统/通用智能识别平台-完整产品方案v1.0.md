# 一、产品概述

## 1.1 产品定位

通用智能识别平台是财运通TMS系统的智能化能力，通过大模型AI技术，支持多种输入方式快速创建运单。

### 核心能力

```
任意格式输入（图片+文字/文字/Excel）
    ↓
AI智能识别与结构化
    ↓
字段智能映射
    ↓
一键创建运单
```

### 核心价值

- ✅ **多输入支持**：图片+文字、纯文字、Excel三种方式
- ✅ **智能识别**：AI自动提取运单字段信息
- ✅ **智能映射**：自动匹配识别字段与系统字段
- ✅ **规则共享**：映射规则全集团共享，越用越准
- ✅ **人工介入**：支持人工确认和调整，确保准确性

---

## 1.4 用户价值

### 核心问题

货运经营者在日常运单录入中面临两个核心痛点：

1. **数据源不一致**：客户通过微信图片、文字消息、邮件Excel等多种渠道发送订单，格式各异，字段命名不统一，调度员需要人工理解和转译。
2. **录入时间成本高**：每张运单需要手动逐字段录入系统，单张耗时3-10分钟，批量订单时效率极低，且容易出错。

### 用户价值

| 价值维度 | 当前状态 | 使用后 |
|----------|----------|--------|
| 录入效率 | 每张运单手动录入3-10分钟 | AI识别+确认，<1分钟完成 |
| 数据准确性 | 人工转录易出错，错误率约5-10% | AI识别+人工校验，准确率≥95% |
| 多格式适配 | 需要人工理解不同格式，重新录入 | 图片/文字/Excel统一识别，无需转译 |
| 字段映射 | 每次手动对应客户字段与系统字段 | 映射规则自动积累，越用越准 |
| 批量处理 | 10张运单需30-100分钟 | 批量上传，统一确认，效率提升80%+ |

### 价值主张

> **让调度员从"数据搬运工"变成"数据确认者"**
>
> 无论客户用什么格式发来订单，系统自动识别、自动映射、自动填单，调度员只需确认和补充关键信息，一键创建运单。

# 七、字段映射机制（通用）

## 7.1 映射规则优先级

```
1. 有映射配置 → 直接应用（不考虑置信度）
2. 无映射配置 + 置信度≥30% → 自动填充
3. 无映射配置 + 置信度<30% → 留空，让用户标记
```

## 7.2 映射匹配算法

```javascript
function findMapping(recognizedField, systemFields, mappingRules) {
  // 1. 查找已保存的映射规则
  const savedRule = mappingRules.find(
    rule => rule.source_field === recognizedField.name
  );
  if (savedRule) {
    return {
      target: savedRule.target_field,
      confidence: 1.0,
      type: 'saved',
      needConfirm: false
    };
  }

  // 2. AI智能匹配（置信度≥30%）
  const aiMatch = aiMatchField(recognizedField.name, systemFields);
  if (aiMatch && aiMatch.confidence >= 0.3) {
    return {
      target: aiMatch.field_id,
      confidence: aiMatch.confidence,
      type: 'ai',
      needConfirm: aiMatch.confidence < 0.9
    };
  }

  // 3. 无匹配：需要用户手动选择
  return {
    target: null,
    confidence: 0,
    type: 'unknown',
    needConfirm: true
  };
}
```

## 7.3 映射冲突处理

### 冲突检测

```javascript
function detectMappingConflict(sourceField, newTargetField, mappingRules) {
  const existingRule = mappingRules.find(
    rule => rule.source_field === sourceField
  );

  if (!existingRule || existingRule.target_field === newTargetField) {
    return { hasConflict: false };
  }

  return {
    hasConflict: true,
    oldMapping: {
      source: sourceField,
      target: existingRule.target_field
    },
    newMapping: {
      source: sourceField,
      target: newTargetField
    }
  };
}
```

### 冲突确认弹窗

```
┌─────────────────────────────────────────────────┐
```
