# 二、用户故事详细拆解

## Epic 1：图片+文字识别

### Story 1.1：单个运单图片识别

**作为** 调度员
**我想要** 上传运单图片并快速识别
**以便于** 快速创建运单，无需手动录入

**验收条件**：
- [ ] 支持上传最多2张图片（JPG/PNG/PDF）
- [ ] 支持拖拽上传
- [ ] 点击"开始识别"后，5秒内返回识别结果
- [ ] 识别准确率 ≥ 95%
- [ ] 显示左右分屏界面：左侧原图，右侧识别结果
- [ ] 识别字段可编辑
- [ ] 映射字段可下拉选择
- [ ] 点击"创建运单"成功创建

**技术要点**：
- 两阶段识别（结构分析 + 数据提取）
- 置信度 ≥ 30% 自动映射
- 置信度 < 30% 不映射，需用户手动选择

---

### Story 1.2：批量运单图片识别

**作为** 调度员
**我想要** 上传多个运单的图片并批量识别
**以便于** 一次性创建多个运单，提高效率

**验收条件**：
- [ ] 支持添加最多10个运单上传区
- [ ] 每个运单最多2张图片
- [ ] 磅单通过时间戳自动判断装货/卸货
- [ ] 普通运单通过字段相似度判断是否同一运单
- [ ] 识别结果以双表头表格展示
- [ ] 支持批量创建运单
- [ ] 显示创建结果（成功/失败）

**技术要点**：
- 磅单时间戳判断：装货时间 < 卸货时间
- 字段相似度判断：匹配度 > 80%

---

### Story 1.3：补充信息处理

**作为** 调度员
**我想要** 输入补充信息（司机、挂车等）
**以便于** 快速补全运单信息

**验收条件**：
- [ ] 支持输入最多1000字符
- [ ] AI自动识别补充信息中的字段
- [ ] 补充信息应用到所有运单
- [ ] 运单已有字段不被覆盖

## Epic 4：字段映射

### Story 4.1：智能字段映射

**作为** 调度员
**我想要** 识别字段自动映射到系统字段
**以便于** 减少手动选择，提高效率

**验收条件**：
- [ ] 有映射规则 → 直接应用
- [ ] 无映射规则 + 置信度≥30% → 自动映射
- [ ] 无映射规则 + 置信度<30% → 不映射，需手动选择
- [ ] 映射字段可下拉选择
- [ ] 按删除键清空映射
- [ ] 批量识别时，映射调整自动应用到所有运单

---

### Story 4.2：映射冲突处理

**作为** 调度员
**我想要** 调整映射关系时，系统提示冲突
**以便于** 决定是否更新全局映射规则

**验收条件**：
- [ ] 检测到映射冲突时弹窗提示
- [ ] 显示原映射和新映射
- [ ] 提供"仅本次生效"和"更新映射规则"选项
- [ ] 选择"更新映射规则"后，全集团生效

---

### Story 4.3：映射规则管理

**作为** 系统管理员
**我想要** 查看和管理映射规则
**以便于** 维护映射规则库

**验收条件**：
- [ ] 入口：系统设置 → 智能识别 → 映射规则管理
- [ ] 显示所有映射规则列表
- [ ] 显示识别字段、系统字段、使用次数
- [ ] 支持搜索映射规则
- [ ] 支持删除映射规则
- [ ] 删除前弹窗确认

---
