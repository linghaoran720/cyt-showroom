## 1. 概述

### 1.1 项目背景

**业务现状**：
客户在物料采购入库时无法指定物料的具体单价，采购单价采用阶梯定价模式（用量越多，单价越低）。需要在一定周期后结算时，根据采购的用量情况计算出具体的价格。

**当前痛点**：
- 入库时无法确定单价，但系统要求必填
- 只能填写预估价格，与实际价格存在差异
- 结算后需要手动调整入库单价和相关费用
- 已发放的物料费用无法自动更新
- 成本核算不准确

### 1.2 项目目标

**核心目标**：
- 支持入库时不填写采购单价
- 结算后可以补录采购总价并自动计算单价
- 自动更新入库单、库存、发放记录的费用
- 提升成本核算准确性

**预期效果**：
- 补录操作时间缩短 80%
- 补录错误率降低 90%
- 成本核算准确率达到 100%

### 1.3 项目范围

**包含（In Scope）**：
- 入库时单价改为选填
- 补录价格功能（单个和批量）
- 物料聚合显示
- 价格输入与自动计算
- 附件上传并同步到入库单
- 自动更新入库单、库存、发放费用
- 补录状态标识和记录追溯

**不包含（Out of Scope）**：
- 不支持修改已补录的价格（如需修改，需要先撤销补录）
- 不涉及报销流程的调整
- 不涉及审批流程

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| 补录价格 | 在入库后，补充录入采购总价并自动计算单价的操作 |
| 物料聚合 | 按物料编码将多个入库单中的同一物料合并显示 |
| 阶梯定价 | 根据采购数量不同，单价不同的定价模式 |
| 价格分配 | 将聚合后的单价分配到各个入库单的过程 |

### 3.2 功能详细说明

#### F001: 入库时单价可为空

**功能描述**：
修改入库登记表单，将采购单价字段从必填改为选填，允许仓管员在无法确定单价时不填写单价直接入库。

**业务规则**：
- 采购单价字段改为选填
- 单价为空时，采购总价也为空
- 保存时不校验单价必填
- 单价为空的入库单，列表页显示"-"

**异常处理**：
- 无特殊异常

**界面示意**：
参考原型：`index.html` - 入库登记表单

---

#### F002: 补录价格入口

**功能描述**：
在入库单列表页增加"补录价格"按钮，支持勾选单个或多个入库单进行补录操作。

**业务规则**：
- 列表页操作栏增加"补录价格"按钮
- 支持勾选单个或多个入库单
- 未勾选入库单时，点击按钮提示"请至少选择一个入库单"
- 显示已选择的入库单数量

**异常处理**：
- 未勾选入库单：提示"请至少选择一个入库单"

**界面示意**：
参考原型：`index.html` - 列表页操作栏

---

#### F003: 批量校验补录条件

**功能描述**：
在打开补录弹窗前，自动校验每个入库单是否满足补录条件，不满足的显示原因，用户可选择继续补录满足条件的入库单。

**业务规则**：
- **条件A**：总价 & 单价都为空 → 可补录
- **条件B**：总价 & 单价不为空 且 未发放 → 可补录
- **限制条件**：
  - 已报销 → 不可补录
  - 已发放（且不满足条件B）→ 不可补录
- 批量勾选时，逐个校验每个入库单
- 不满足的显示原因列表
- 提示"X个入库单可补录，Y个不满足条件，是否继续？"
- 用户可选择继续补录满足条件的入库单

**异常处理**：
- 所有入库单都不满足条件：提示"所选入库单均不满足补录条件"，不打开弹窗

**界面示意**：
参考原型：`index.html` - 批量校验提示

---

#### F004: 物料聚合显示

**功能描述**：
按物料编码聚合显示物料信息，显示物料编码、物料名称、总数量、来源入库单数量。

**业务规则**：
- 按物料编码聚合显示
- 显示物料编码、物料名称、物料单位
- 显示总数量（所有入库单的数量之和）
- 显示来源入库单数量（如"来自3个入库单"）
- 物料编码相同但来自不同入库单的，合并为一条记录

**异常处理**：
- 无特殊异常

**界面示意**：
参考原型：`index.html` - 物料聚合显示区域

---

#### F005: 价格输入与自动计算

**功能描述**：
支持输入单价自动计算总价，或输入总价自动计算单价，实时联动计算。

**业务规则**：
- 输入单价 → 总价 = 单价 × 总数量
- 输入总价 → 单价 = 总价 ÷ 总数量
- 单价和总价保留2位小数，四舍五入
- 实时联动计算，无需点击按钮

**异常处理**：
- 输入非数字：提示"请输入有效的数字"
- 输入负数或0：提示"价格必须大于0"

**界面示意**：
参考原型：`index.html` - 价格输入区域

---

#### F006: 部分物料补录

**功能描述**：
支持只补录部分物料的价格，用户可以勾选需要补录的物料。

**业务规则**：
- 每个物料前有复选框，默认全部勾选
- 可以取消勾选不需要补录的物料
- 未勾选的物料不会被补录价格
- 至少勾选一个物料且填写价格，才能提交
- 汇总信息只统计已勾选的物料

**异常处理**：
- 未勾选任何物料或未填写价格：提示"请至少为一个物料补录价格"

**界面示意**：
参考原型：`index.html` - 物料卡片选择

---

#### F007: 附件上传

**功能描述**：
补录价格时可以上传账单附件，支持 JPG、PNG、PDF 格式。

**业务规则**：
- 补录弹窗中提供附件上传入口
- 支持上传 JPG、PNG、PDF 格式文件
- 单个文件大小不超过 10MB
- 可以上传多个文件
- 显示已上传文件列表（文件名、大小）
- 可以删除已上传的文件

**异常处理**：
- 文件格式不支持：提示"仅支持 JPG、PNG、PDF 格式"
- 文件过大：提示"文件大小不能超过 10MB"
- 上传失败：提示"附件上传失败，请重试"，但不影响价格补录

**界面示意**：
参考原型：`index.html` - 附件上传区域

---

#### F008: 附件同步到入库单

**功能描述**：
补录价格时上传的附件，自动同步到所有被补录的入库单。

**业务规则**：
- 补录价格时上传的附件，同步到所有被补录的入库单
- 批量补录时，附件同步到所有满足条件的入库单
- 入库单详情页可以查看补录时上传的附件
- 附件URL保存为JSON数组格式
- 同一批补录的入库单，附件列表完全相同

**异常处理**：
- 附件同步失败：回滚整个补录操作

**数据结构**：
```json
{
  "price_fill_files": [
    {"name": "账单.pdf", "url": "https://..."},
    {"name": "发票.jpg", "url": "https://..."}
  ]
}
```

#### F009: 价格分配到入库单

**功能描述**：
将聚合后的单价自动分配到各个入库单，更新每个入库单的采购单价和总价。

**业务规则**：
- 按物料编码将单价分配到对应的入库单
- 每个入库单的总价 = 单价 × 该入库单的数量
- 更新入库单的 `buy_unit_price` 和 `buy_total_price` 字段
- 更新入库单的补录状态字段 `is_price_filled` = 1
- 记录补录操作人和补录时间
- 使用数据库事务确保数据一致性

**异常处理**：
- 任何一个入库单更新失败：回滚所有操作，提示"补录失败，请重试"

---

#### F010: 更新库存单价

**功能描述**：
补录价格后自动更新库存表的单价和总价。

**业务规则**：
- 补录价格后，更新库存表的 `buy_unit_price` 字段
- 更新库存表的 `buy_total_price` 字段
- 按入库单ID和物料编码匹配库存记录
- 使用数据库事务确保数据一致性

**异常处理**：
- 库存记录不存在：记录日志，继续执行
- 更新失败：回滚所有操作

---

#### F011: 更新发放费用

**功能描述**：
补录价格后自动更新所有发放记录的费用。

**业务规则**：
- 查询入库单关联的所有发放记录
- 按新单价重新计算发放费用（发放总价 = 新单价 × 发放数量）
- 更新发放记录的 `unit_price` 和 `total_price` 字段
- 发放总价保留2位小数，四舍五入
- 使用数据库事务确保数据一致性
- 超过100条发放记录时，使用异步处理

**异常处理**：
- 发放记录不存在：正常，无需处理
- 更新失败：回滚所有操作

---

#### F012: 补录日志记录

**功能描述**：
记录每次补录操作的详细日志，用于追溯价格变动历史和审计。

**业务规则**：
- 记录补录的入库单ID列表
- 记录物料编码、物料名称、总数量
- 记录原单价、新单价、原总价、新总价
- 记录补录操作人和补录时间
- 记录上传的附件列表
- 记录补录备注（如有）

**异常处理**：
- 日志记录失败：记录系统日志，但不影响补录操作

**数据表**：
- 表名：`MaterialPriceFillLog`
- 每次补录操作，每个物料生成一条日志记录

**功能描述**：
在补录弹窗中显示汇总信息，包括补录物料数和总金额。

**业务规则**：

- 显示补录总金额（所有已勾选物料的总价之和）

#### F015: 补录成功反馈

**功能描述**：
补录成功后显示明确的成功提示。

**异常处理**：
- 无特殊异常

**界面示意**：
参考原型：`index.html` - 成功提示弹窗

### 3.3 非功能需求

**性能要求**：
- 补录价格操作响应时间 < 3秒
- 批量更新发放费用时，支持异步处理（超过100条记录）
- 物料聚合查询响应时间 < 2秒
- 附件上传大小限制：单个文件 ≤ 10MB

**安全要求**：
- 补录价格操作需要权限控制（权限码：`materialPriceFill`）
- 已报销的入库单不允许补录价格
- 补录操作记录日志，可追溯
- 附件上传需要校验文件类型和大小
- 使用数据库事务确保数据一致性

**兼容性要求**：
- 浏览器：Chrome 90+、Safari 14+
- 兼容现有入库流程
- 已有入库单不受影响

## 5. 数据需求

### 5.1 数据模型

**表名：MaterialInboundBill（物料入库单表）**

新增字段：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| is_price_filled | TINYINT | 是 | 是否已补录价格：0-未补录，1-已补录 |
| price_fill_user | VARCHAR(50) | 否 | 补录操作人 |
| price_fill_time | DATETIME | 否 | 补录时间 |
| price_fill_remark | VARCHAR(500) | 否 | 补录备注 |
| price_fill_files | TEXT | 否 | 补录附件URL列表，JSON格式 |

**表名：MaterialPriceFillLog（补录日志表）**

新建表：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键ID |
| bill_ids | VARCHAR(500) | 是 | 入库单ID列表，逗号分隔 |
| material_code | VARCHAR(50) | 是 | 物料编码 |
| material_name | VARCHAR(100) | 是 | 物料名称 |
| total_quantity | DECIMAL(10,3) | 是 | 总数量 |
| old_unit_price | DECIMAL(10,2) | 否 | 原单价 |
| new_unit_price | DECIMAL(10,2) | 是 | 新单价 |
| old_total_price | DECIMAL(10,2) | 否 | 原总价 |
| new_total_price | DECIMAL(10,2) | 是 | 新总价 |
| fill_user | VARCHAR(50) | 是 | 补录操作人 |
| fill_time | DATETIME | 是 | 补录时间 |
| fill_files | TEXT | 否 | 补录附件URL列表，JSON格式 |
| remark | VARCHAR(500) | 否 | 备注 |
| created_at | DATETIME | 是 | 创建时间 |

### 5.2 数据流转

```
1. 入库登记（单价为空）
   ↓
2. 物料发放（使用物料维护表的单价）
   ↓
3. 结算后补录价格
   ↓
4. 更新入库单价格
   ↓
5. 更新库存单价
   ↓
6. 更新发放费用
   ↓
7. 附件同步到入库单
   ↓
8. 记录补录日志
```

## 7. 验收标准

### 7.1 功能验收

- [ ] 入库时可以不填写采购单价，系统正常保存
- [ ] 补录价格前，系统自动校验入库单是否满足条件
- [ ] 批量补录时，显示不满足条件的入库单和原因
- [ ] 物料按编码聚合显示，显示总数量和来源入库单数量
- [ ] 输入单价或总价时，另一个值自动计算
- [ ] 可以只补录部分物料的价格
- [ ] 可以上传账单附件，附件同步到所有入库单
- [ ] 补录价格后，入库单、库存、发放记录的费用自动更新
- [ ] 已报销的入库单不允许补录价格，系统提示错误
- [ ] 补录操作记录日志，可追溯操作历史
- [ ] 列表页可以按补录状态筛选入库单

### 7.2 性能验收

- [ ] 补录价格操作响应时间 < 3秒
- [ ] 物料聚合查询响应时间 < 2秒
- [ ] 批量更新100条发放记录，响应时间 < 5秒

### 7.3 安全验收

- [ ] 补录价格操作需要权限，无权限用户无法操作
- [ ] 已报销的入库单不允许补录价格
- [ ] 补录操作记录日志，包含操作人、操作时间、价格变化
- [ ] 附件上传校验文件类型和大小
