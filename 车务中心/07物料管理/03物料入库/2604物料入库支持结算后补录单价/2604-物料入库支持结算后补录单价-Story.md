# 用户故事集

## Epic: 物料入库支持结算后补录单价

### Story 1: 入库时单价可为空

**用户故事**
作为 仓管员
我想要 在物料入库时可以不填写采购单价
以便 适应阶梯定价的采购场景，避免填写不准确的预估价格

**优先级**：P0

**故事点**：3

**验收标准**
- [ ] AC1: 入库登记表单中，采购单价字段改为选填（非必填）
- [ ] AC2: 单价为空时，采购总价也为空，系统正常保存
- [ ] AC3: 单价为空的入库单，列表页显示"-"，不显示0或空白
- [ ] AC4: 单价为空的入库单，补录状态显示"未补录"

**数据埋点**
- 事件：入库登记（单价为空）
- 关键指标：单价为空的入库单占比

---

### Story 2: 补录价格入口

**用户故事**
作为 财务人员
我想要 在入库单列表页有明确的补录价格入口
以便 快速找到并执行补录操作

**优先级**：P0

**故事点**：2

**验收标准**
- [ ] AC1: 列表页操作栏增加"补录价格"按钮
- [ ] AC2: 支持勾选单个或多个入库单
- [ ] AC3: 未勾选入库单时，点击按钮提示"请至少选择一个入库单"
- [ ] AC4: 显示已选择的入库单数量

**技术说明**
- 按钮权限：`materialPriceFill`

---

### Story 3: 批量校验补录条件

**用户故事**
作为 财务人员
我想要 在补录价格前系统自动校验入库单是否满足条件
以便 避免对不符合条件的入库单进行补录操作

**优先级**：P0

**故事点**：5

**验收标准**
- [ ] AC1: 校验条件A：总价 & 单价都为空 → 可补录
- [ ] AC2: 校验条件B：总价 & 单价不为空 且 未发放 → 可补录
- [ ] AC3: 已报销的入库单不允许补录，提示"已报销，不可补录"
- [ ] AC4: 已发放（且不满足条件B）的入库单不允许补录，提示"已发放，不可补录"
- [ ] AC5: 批量勾选时，显示不满足条件的入库单列表和原因
- [ ] AC6: 提示"X个入库单可补录，Y个不满足条件，是否继续？"
- [ ] AC7: 用户可选择继续补录满足条件的入库单

**业务规则**
- 校验在打开补录弹窗前执行
- 如果所有入库单都不满足条件，不打开弹窗，直接提示

### Story 4: 物料聚合显示

**用户故事**
作为 财务人员
我想要 看到按物料编码聚合后的物料信息
以便 清楚了解需要补录价格的物料总量和来源

**优先级**：P0

**故事点**：5

**验收标准**
- [ ] AC1: 按物料编码聚合显示物料信息
- [ ] AC2: 显示物料编码、物料名称、物料单位
- [ ] AC3: 显示总数量（所有入库单的数量之和）
- [ ] AC4: 显示来源入库单数量（如"来自3个入库单"）
- [ ] AC5: 物料编码相同但来自不同入库单的，合并为一条记录

**技术说明**
- 聚合算法：按物料编码分组，累加数量
- 记录每个物料对应的入库单列表，用于后续价格分配

### Story 5: 价格输入与自动计算

**用户故事**
作为 财务人员
我想要 输入单价或总价时，系统自动计算另一个值
以便 减少手动计算，避免计算错误

**优先级**：P0

**故事点**：3

**验收标准**
- [ ] AC1: 输入单价后，总价自动计算（总价 = 单价 × 总数量）
- [ ] AC2: 输入总价后，单价自动计算（单价 = 总价 ÷ 总数量）
- [ ] AC3: 单价和总价保留2位小数，四舍五入
- [ ] AC4: 输入非数字时，提示"请输入有效的数字"
- [ ] AC5: 输入负数或0时，提示"价格必须大于0"
- [ ] AC6: 实时联动计算，无需点击按钮

**UI/UX 说明**
- 单价和总价输入框并排显示，中间用"或"字分隔
- 输入框获得焦点时，清空另一个输入框的自动计算值（可选）

---

### Story 6: 部分物料补录

**用户故事**
作为 财务人员
我想要 可以只补录部分物料的价格
以便 灵活处理不同物料的结算时间差异

**优先级**：P1

**故事点**：3

**验收标准**
- [ ] AC1: 每个物料前有复选框，默认全部勾选
- [ ] AC2: 可以取消勾选不需要补录的物料
- [ ] AC3: 未勾选的物料不会被补录价格
- [ ] AC4: 至少勾选一个物料且填写价格，才能提交
- [ ] AC5: 汇总信息只统计已勾选的物料

**业务规则**
- 部分补录后，入库单的补录状态仍为"未补录"
- 只有所有物料都补录后，状态才变为"已补录"

### Story 7: 附件上传

**用户故事**
作为 财务人员
我想要 在补录价格时上传账单附件
以便 保留结算凭证，方便后续审计和追溯

**优先级**：P1

**故事点**：3

**验收标准**
- [ ] AC1: 补录弹窗中提供附件上传入口
- [ ] AC2: 支持上传 JPG、PNG、PDF 格式文件
- [ ] AC3: 单个文件大小不超过 10MB
- [ ] AC4: 可以上传多个文件
- [ ] AC5: 显示已上传文件列表（文件名、大小）
- [ ] AC6: 可以删除已上传的文件
- [ ] AC7: 附件上传失败时，提示错误信息，但不影响价格补录

**技术说明**
- 附件上传到文件服务器（阿里云OSS或本地）
- 附件URL保存到入库单的 `price_fill_files` 字段（JSON格式）

---

### Story 8: 附件同步到入库单

**用户故事**
作为 财务人员
我想要 上传的账单附件自动同步到所有相关入库单
以便 每个入库单都能查看到结算凭证

**优先级**：P1

**故事点**：3

**验收标准**
- [ ] AC1: 补录价格时上传的附件，同步到所有被补录的入库单
- [ ] AC2: 批量补录时，附件同步到所有满足条件的入库单
- [ ] AC3: 入库单详情页可以查看补录时上传的附件
- [ ] AC4: 附件URL保存为JSON数组格式
- [ ] AC5: 同一批补录的入库单，附件列表完全相同

**数据结构**
```json
{
  "price_fill_files": [
    {"name": "账单.pdf", "url": "https://..."},
    {"name": "发票.jpg", "url": "https://..."}
  ]
}
```

### Story 9: 价格分配到入库单

**用户故事**
作为 系统
我想要 将聚合后的单价自动分配到各个入库单
以便 更新每个入库单的采购单价和总价

**优先级**：P0

**故事点**：5

**验收标准**
- [ ] AC1: 按物料编码将单价分配到对应的入库单
- [ ] AC2: 每个入库单的总价 = 单价 × 该入库单的数量
- [ ] AC3: 更新入库单的 `buy_unit_price` 和 `buy_total_price` 字段
- [ ] AC4: 更新入库单的补录状态字段 `is_price_filled` = 1
- [ ] AC5: 记录补录操作人和补录时间
- [ ] AC6: 使用数据库事务确保数据一致性

**技术说明**
- 分配算法：遍历物料对应的入库单列表，逐个更新
- 事务回滚：任何一个入库单更新失败，回滚所有操作

以便 保持库存数据与入库单数据一致

**优先级**：P0

**故事点**：3

**验收标准**

### Story 11: 更新发放费用

**用户故事**
作为 系统
我想要 补录价格后自动更新所有发放记录的费用
以便 确保发放费用与实际采购价格一致

**优先级**：P0

**故事点**：8

**验收标准**
- [ ] AC1: 查询入库单关联的所有发放记录
- [ ] AC2: 按新单价重新计算发放费用（发放总价 = 新单价 × 发放数量）
- [ ] AC3: 更新发放记录的 `unit_price` 和 `total_price` 字段
- [ ] AC4: 发放总价保留2位小数，四舍五入
- [ ] AC5: 使用数据库事务确保数据一致性
- [ ] AC6: 超过100条发放记录时，使用异步处理

**技术说明**
- 发放表：`MaterialApply`
- 匹配条件：`inbound_bill_id` + `material_code`
- 异步处理：使用消息队列或后台任务

以便 追溯价格变动历史和审计

**优先级**：P1

**故事点**：3

**验收标准**
- [ ] AC1: 记录补录的入库单ID列表
