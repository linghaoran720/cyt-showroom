# 需求分析报告 — 物料盘点

## 1. 需求概述
- **需求名称**：库存盘点
- **需求类型**：新功能
- **需求来源**：客户
- **优先级建议**：P1高
- **所属模块**：物料管理 - 库存管理
- **知识库参考**：`knowledge-base/物料管理/物料管理知识.html`

## 2. 业务背景
- **当前痛点**：物料在实际使用中会出现丢失、破损、计量误差等情况，系统记录的可用库存与仓库实际数量产生偏差，但目前系统没有修正机制
- **期望目标**：客户完成实物盘点后，能在系统中录入盘点结果，系统自动计算差异并调整库存数量
- **业务价值**：
  - 保证库存数据准确性，避免"有单无货"或"有货不知"
  - 为物料申请、发放提供可靠的库存基础
  - 支撑财务核算物料损耗和资产盘亏/盘盈

## 3. 用户角色与场景

| 角色 | 场景描述 | 核心诉求 |
|------|----------|----------|
| 仓管员 | 月末/季末对仓库物料进行实物清点，将盘点结果录入系统 | 快速录入盘点数量，系统自动算出差异 |
| 车队管理员 | 查看盘点结果，了解物料损耗情况 | 掌握各网点物料损耗，控制异常 |
| 财务人员 | 根据盘点差异核算物料损耗金额 | 获取盘盈/盘亏的金额数据 |

## 4. 功能需求拆解

### 4.1 核心功能
- [ ] **创建盘点单**：选择盘点网点，自动带出该网点下的物料及系统当前库存数量
- [ ] **录入盘点数量**：逐项填写实际盘点数量，系统自动计算差异（盘盈/盘亏）
- [ ] **提交校验**：提交时校验库存减少项，当前库存 ≥ 减少数量才允许提交，否则提示具体物料及差额
- [ ] **库存调整执行**：提交通过后直接更新物料的当前库存和可用库存
- [ ] **入库单关联 — 盘亏扣减**：盘亏时按FIFO原则关联入库单，从最早创建且仍有剩余库存的入库单开始扣减，单价取该入库单的入库单价，自动计算盘亏金额（扣减数量 × 入库单价）；若单条入库单余量不足，继续扣减下一条入库单，直至扣完
- [ ] **入库单关联 — 盘盈入库**：盘盈时自动生成一条入库单，来源标记为「盘点」，入库单价取该物料最近一次入库单价，入库数量为盘盈数量
- [ ] **撤销盘点**：已提交的盘点单可撤销，撤销时校验库存可恢复性（若相关物料已被领用导致库存不足则拦截），撤销后库存恢复至盘点前状态，盘点单变为「已撤销」
- [ ] **撤销 — 入库单联动**：撤销时，盘亏项恢复对应入库单的剩余库存；盘盈项作废对应的盘点入库单
- [ ] **重新编辑**：已撤销的盘点单可重新编辑并再次提交，重新提交时重新执行入库单关联逻辑

### 4.2 辅助功能
- [ ] **盘点单列表**：查看历史盘点记录，支持按网点、时间、状态筛选
- [ ] **盘点差异报表**：汇总盘盈/盘亏明细，支持导出
- [ ] **批量导入盘点数据**：支持Excel导入盘点结果

### 4.3 状态流转
- 盘点单状态：草稿 → 已提交 → 已撤销
- 草稿：可编辑、可删除、可提交
- 已提交：可查看、可撤销
- 已撤销：可编辑（重新提交）、可删除

### 4.3 权限与网点规则
- [ ] **功能权限**：仅拥有"物料盘点"功能权限的用户可见和操作盘点功能
- [ ] **网点权限**：用户只能对本级网点或下级网点发起盘点，不能跨级操作非管辖网点

### 4.4 非功能需求
- **数据一致性**：库存调整必须在事务中完成，确保不出现中间状态
- **操作日志**：所有库存调整需记录操作日志，可追溯

## 5. 边界与约束

- **包含（In Scope）**：
  - 盘点单的创建、录入、提交、撤销、重新编辑全流程
  - 盘点导致的库存数量调整
  - 盘亏关联入库单（FIFO扣减）及金额计算
  - 盘盈自动生成入库单（来源：盘点）
  - 撤销时入库单联动恢复/作废
  - 盘点差异报表
  - 按网点维度盘点，网点层级权限控制
- **不包含（Out of Scope）**：
  - 移动端盘点
  - 条码/RFID扫码盘点
  - 审批流程
  - 盘点期间冻结机制
  - 差异阈值预警

## 6. 风险评估

| 风险项 | 影响程度 | 应对策略 |
|--------|----------|----------|
| 撤销时相关物料已被领用导致库存不足 | 高 | 撤销时实时校验库存可恢复性，不满足则拦截并提示具体物料 |
| 提交时库存已被其他操作变更 | 高 | 提交时实时校验当前库存，不满足则拦截并提示 |
| 盘点数量录入错误 | 中 | 差异列醒目标记，提交前展示汇总确认 |
| 大量物料盘点录入效率低 | 中 | 支持Excel批量导入 |

## 7. 验收标准

- [ ] 可创建盘点单，选择网点后自动带出该网点物料及系统库存数量
- [ ] 可逐项录入实际盘点数量，系统自动计算差异并标记盘盈/盘亏
- [ ] 提交时校验：库存减少项的当前库存 ≥ 减少数量，否则拦截提示
- [ ] 提交后系统库存自动更新，物料库存列表中数值正确
- [ ] 盘亏按FIFO关联入库单扣减，盘亏金额 = 扣减数量 × 对应入库单价
- [ ] 盘盈自动生成来源为「盘点」的入库单，单价取最近一次入库单价
- [ ] 盘点单列表支持按网点、状态、时间筛选
- [ ] 支持导出盘点差异报表
- [ ] 已提交的盘点单可撤销，撤销后库存恢复，盘点单状态变为「已撤销」
- [ ] 撤销时校验：若相关物料已被领用导致库存不足，撤销被拦截并提示
- [ ] 撤销时盘亏项恢复对应入库单库存，盘盈项作废对应的盘点入库单
- [ ] 已撤销的盘点单可重新编辑并再次提交
- [ ] 仅有"物料盘点"权限的用户可操作，网点权限限制生效
- [ ] 所有库存调整操作有日志记录
