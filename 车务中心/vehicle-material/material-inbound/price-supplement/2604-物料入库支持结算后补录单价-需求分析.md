# 需求分析报告

## 1. 需求概述

- **需求名称**：物料入库支持结算后补录单价
- **需求类型**：功能优化
- **需求来源**：客户业务场景需求
- **优先级建议**：P1 高

## 2. 业务背景

### 2.1 当前痛点

**现状问题**：
- 当前系统要求物料入库时必须填写采购单价（必填字段）
- 客户实际业务中，采购单价无法在入库时确定
- 采购单价采用阶梯定价模式：用量越多，单价越低
- 需要在一定周期后结算时，根据实际用量计算出具体价格

**业务影响**：
- 入库时无法准确填写单价，只能填写预估价格
- 结算后实际价格与预估价格存在差异
- 需要手动调整入库单价和相关费用，工作量大且容易出错
- 已发放的物料费用无法自动更新，导致成本核算不准确

### 2.2 期望目标

- 支持入库时不填写采购单价（单价字段改为选填）
- 结算后可以补录采购总价
- 系统自动计算单价（采购总价 ÷ 入库数量）
- 自动更新入库单的费用信息
- 自动更新已发放物料的费用信息

### 2.3 业务价值

- **提升准确性**：避免预估价格与实际价格的差异
- **降低工作量**：减少手动调整费用的工作
- **提高效率**：自动计算和更新费用，节省时间
- **成本核算准确**：确保物料成本数据的准确性

## 3. 用户角色与场景

| 角色 | 场景描述 | 核心诉求 |
|------|----------|----------|
| 仓管员 | 物料采购入库时，无法确定单价 | 希望可以不填单价直接入库 |
| 财务人员 | 结算时根据用量计算出实际价格 | 希望可以补录总价并自动计算单价 |
| 财务人员 | 需要更新已发放物料的费用 | 希望系统自动更新相关费用 |

## 4. 功能需求拆解

### 4.1 核心功能

- [ ] **功能1：入库时单价改为选填**
  - 修改入库登记表单，采购单价字段改为选填
  - 单价为空时，采购总价也为空
  - 保存时不校验单价必填

- [ ] **功能2：补录采购总价**
  - 新增"补录价格"操作按钮
  - 打开补录价格弹窗，显示入库单信息
  - 填写采购总价（必填）
  - 系统自动计算单价 = 采购总价 ÷ 入库数量
  - 保存后更新入库单的采购单价和采购总价

- [ ] **功能3：自动更新入库费用**
  - 补录价格后，更新入库单的采购总价
  - 更新物料库存表的采购单价和采购总价

- [ ] **功能4：自动更新发放费用**
  - 查询该入库单关联的所有物料发放记录
  - 根据新的单价，重新计算发放费用
  - 更新物料发放表的单价和总价

### 4.2 辅助功能

- [ ] **功能5：补录状态标识**
  - 入库单增加"是否已补录价格"字段
  - 列表页显示补录状态（已补录/未补录）
  - 支持按补录状态筛选

- [ ] **功能6：补录记录追溯**
  - 记录补录操作人和补录时间
  - 记录补录前后的价格变化
  - 支持查看补录历史

### 4.3 非功能需求

- **性能要求**：
  - 补录价格操作响应时间 < 3秒
  - 批量更新发放费用时，支持异步处理

- **安全要求**：
  - 补录价格操作需要权限控制
  - 已报销的入库单不允许补录价格
  - 补录操作记录日志

- **兼容性要求**：
  - 兼容现有入库流程
  - 已有入库单不受影响

## 5. 边界与约束

### 5.1 包含（In Scope）

- 入库时单价改为选填
- 结算后补录采购总价
- 自动计算单价
- 自动更新入库费用和发放费用
- 补录状态标识和记录追溯

### 5.2 不包含（Out of Scope）

- 不支持修改已补录的价格（如需修改，需要先撤销补录）
- 不支持批量补录多个入库单
- 不涉及报销流程的调整

### 5.3 依赖项

- 需要物料库存模块配合更新库存单价
- 需要物料发放模块配合更新发放费用
- 需要权限管理模块配置补录价格权限

## 6. 风险评估

| 风险项 | 影响程度 | 应对策略 |
|--------|----------|----------|
| 已发放物料数量与入库数量不一致 | 高 | 按比例分摊费用，确保总费用一致 |
| 补录价格后报销单已生成 | 高 | 已报销的入库单不允许补录价格 |
| 补录操作失败导致数据不一致 | 高 | 使用事务确保数据一致性 |
| 历史数据兼容性 | 中 | 历史数据保持不变，新功能仅对新数据生效 |

## 7. 验收标准

- [ ] 入库时可以不填写采购单价，系统正常保存
- [ ] 补录价格后，系统自动计算单价并更新入库单
- [ ] 补录价格后，关联的物料发放记录费用自动更新
- [ ] 已报销的入库单不允许补录价格，系统提示错误
- [ ] 补录操作记录日志，可追溯操作历史
- [ ] 列表页可以按补录状态筛选入库单

## 8. 业务规则

### 8.1 补录价格校验规则

1. **状态检查**：
   - 只能对"登记入库"类型的记录补录价格
   - 已报销的入库单不允许补录价格
   - 已补录价格的入库单不允许重复补录

2. **数据校验**：
   - 采购总价必须大于0
   - 采购总价必须是数字类型，支持2位小数

3. **权限检查**：
   - 需要有"补录价格"权限

### 8.2 费用计算规则

1. **单价计算**：
   - 单价 = 采购总价 ÷ 入库数量
   - 单价保留2位小数，四舍五入

2. **发放费用更新**：
   - 发放总价 = 新单价 × 发放数量
   - 发放总价保留2位小数，四舍五入

3. **费用分摊**：
   - 如果入库数量与已发放数量不一致，按比例分摊
   - 确保所有发放费用之和 ≤ 采购总价

## 9. 数据影响范围

### 9.1 涉及的数据表

- **MaterialInboundBill**（物料入库单表）
  - 新增字段：`is_price_filled`（是否已补录价格）
  - 新增字段：`price_fill_user`（补录操作人）
  - 新增字段：`price_fill_time`（补录时间）

- **MaterialStock**（物料库存表）
  - 更新字段：`buy_unit_price`（采购单价）
  - 更新字段：`buy_total_price`（采购总价）

- **MaterialApply**（物料发放表）
  - 更新字段：`unit_price`（单价）
  - 更新字段：`total_price`（总价）

### 9.2 数据流转

```
1. 入库登记（单价为空）
   ↓
2. 物料发放（使用物料维护表的单价）
   ↓
3. 结算后补录价格
   ↓
4. 更新入库单价格
   ↓
5. 更新库存单价
   ↓
6. 更新发放费用
```

## 10. 后续建议

1. **一期实现**（MVP）：
   - 入库时单价改为选填
   - 补录采购总价并自动计算单价
   - 自动更新入库费用和发放费用

2. **二期优化**：
   - 支持批量补录多个入库单
   - 支持修改已补录的价格
   - 补录价格时支持按物料明细分别补录

3. **三期扩展**：
   - 支持价格审批流程
   - 支持价格变动预警
   - 支持价格趋势分析
