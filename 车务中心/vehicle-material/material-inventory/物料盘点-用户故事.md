# 用户故事集 — 物料盘点

## Epic: 物料盘点

### Story 1: 创建盘点单

**用户故事**
```
作为 仓管员
我想要 选择网点后创建盘点单，系统自动带出该网点所有物料及当前库存数量
以便 快速开始盘点工作，不需要手动查找物料信息
```

**优先级**：P0
**故事点估算**：5

**验收标准**
- [ ] AC1: 创建盘点单时可选择盘点网点，仅显示本级和下级网点
- [ ] AC2: 选择网点后自动加载该网点下所有物料，显示物料编码、名称、类别、规格、单位、系统当前库存
- [ ] AC3: 无"物料盘点"功能权限的用户不可见该功能入口

**技术说明**
- 网点权限需对接现有组织架构，校验用户与网点的层级关系
- 物料列表数据来源于物料库存模块

### Story 2: 录入盘点数量并自动计算差异

**用户故事**
```
作为 仓管员
我想要 逐项填写实际盘点数量，系统自动计算出与系统库存的差异
以便 清楚看到每项物料的盘盈或盘亏情况
```

**优先级**：P0
**故事点估算**：3

**验收标准**
- [ ] AC1: 每项物料可输入"实际盘点数量"，输入后自动计算差异 = 盘点数量 - 系统库存
- [ ] AC2: 差异为正数标记为"盘盈"（绿色），负数标记为"盘亏"（红色），零标记为"一致"
- [ ] AC3: 盘点数量仅允许输入非负整数
- [ ] AC4: 支持在列表中快速逐行录入，Tab键可跳转下一行

### Story 4: 提交盘点单并校验库存

**用户故事**
```
作为 仓管员
我想要 提交盘点单时系统自动校验库存减少项的合法性
以便 确保不会出现库存扣成负数的情况
```

**优先级**：P0
**故事点估算**：5

**验收标准**
- [ ] AC1: 提交时校验所有盘亏项：当前系统库存 ≥ 减少数量，否则拦截
- [ ] AC2: 校验不通过时，提示具体物料名称、系统库存、盘点数量、差额
- [ ] AC3: 提交前展示盘点汇总（盘盈N项、盘亏N项、一致N项、盘亏总金额、盘盈总金额），用户确认后提交
- [ ] AC4: 提交成功后自动更新物料的当前库存和可用库存
- [ ] AC5: 盘亏项按FIFO关联入库单扣减——从最早创建且仍有剩余库存的入库单开始扣，单条不够则继续扣下一条，记录每条入库单的扣减数量和单价
- [ ] AC6: 盘亏金额 = 各入库单扣减数量 × 对应入库单价之和
- [ ] AC7: 盘盈项自动生成一条入库单，来源标记为「盘点」，入库数量为盘盈数量，入库单价取该物料最近一次入库单价
- [ ] AC8: 库存调整和入库单操作在同一事务中完成，全部成功或全部回滚

**技术说明**
- 提交时需实时查询当前库存（非创建盘点单时的快照），防止并发操作导致数据不一致
- 盘亏扣减需记录关联的入库单ID和扣减明细，用于撤销时恢复
- 盘盈生成的入库单需记录关联的盘点单ID，用于撤销时作废

### Story 8: 撤销已提交的盘点单

**用户故事**
```
作为 仓管员
我想要 撤销已提交的盘点单，系统自动恢复库存至盘点前状态
以便 在发现盘点数据有误时能够纠正，而不是只能将错就错
```

**优先级**：P0
**故事点估算**：5

**验收标准**
- [ ] AC1: 已提交状态的盘点单，在列表和详情页均可执行「撤销」操作
- [ ] AC2: 撤销前弹出确认弹窗，提示"撤销后库存将恢复至盘点前状态"
- [ ] AC3: 撤销时校验库存可恢复性——若盘盈物料在盘点后已被领用导致当前库存不足以扣回，则拦截撤销并提示具体物料及差额
- [ ] AC4: 撤销成功后，盘亏项恢复对应入库单的剩余库存（按提交时记录的扣减明细反向恢复）
- [ ] AC5: 撤销成功后，盘盈项作废对应的盘点入库单（来源为「盘点」的入库单）
- [ ] AC6: 物料库存恢复，盘点单状态变为「已撤销」
- [ ] AC7: 撤销操作记录到操作日志

**技术说明**
- 撤销本质是反向库存调整：盘盈项扣减库存并作废入库单、盘亏项增加库存并恢复入库单余量
- 需在事务中完成，确保原子性
- 依赖提交时记录的入库单扣减明细进行精确恢复

### Story 9: 重新编辑已撤销的盘点单

**用户故事**
```
作为 仓管员
我想要 对已撤销的盘点单重新编辑并再次提交
以便 修正盘点数据后重新生效，不需要从头创建新的盘点单
```

**优先级**：P1
**故事点估算**：3

**验收标准**
- [ ] AC1: 已撤销状态的盘点单可点击「编辑」进入编辑页面
- [ ] AC2: 编辑页面加载该盘点单原有数据，系统库存刷新为当前最新值
- [ ] AC3: 修改盘点数量后可重新提交，提交逻辑与新建盘点一致（含库存校验）
- [ ] AC4: 重新提交后状态变为「已提交」

---

## 状态流转说明

```
草稿 ──提交──→ 已提交 ──撤销──→ 已撤销
 ↑                                  │
 └──────── 重新编辑提交 ←───────────┘
```

- 草稿：可编辑、可删除、可提交
- 已提交：可查看、可撤销
- 已撤销：可编辑（重新提交）、可删除
