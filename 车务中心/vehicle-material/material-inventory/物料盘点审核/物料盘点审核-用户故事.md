# 用户故事集 — 物料盘点审核

## Epic: 物料盘点审核

### 需求背景

当前物料盘点流程为：仓管员创建盘点单 → 录入盘点数量 → 提交后直接调整库存。由于登记员（仓管员）录入错误后可频繁撤销、修改、重新提交，缺乏审核环节，导致库存数据频繁波动，难以追溯责任。

本次在现有盘点功能基础上增加「审批」和「取消审批」功能，提交后不再直接生效，而是进入审批环节，由审批人确认后才执行库存调整。审批的具体流程配置由独立的审批模块负责，本次仅对接审批结果。

### 状态流转变更

**原状态流转**：
```
草稿 ──提交──→ 已提交 ──撤销──→ 已撤销
```

**新状态流转**：
```
草稿 ──提交──→ 待审批 ──审批流转──→ 审批中 ──审批通过──→ 已通过（库存生效）──撤销──→ 已撤销
                 │                    │
                 │                    ├──审批驳回──→ 已驳回 ──修改保存──→ 审批中（恢复原审批节点）
                 │                    │
                 │                    └──（继续流转下一级审批）
                 │
                 └──取消审批──→ 草稿（可重新编辑提交）
```

**状态说明**：
| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| 草稿 | 新建或取消审批后回到草稿 | 编辑、删除、提交 |
| 待审批 | 已提交等待第一级审批人处理 | 取消审批（提交人） |
| 审批中 | 审批流程已开始流转，可能涉及多级审批 | 查看审批进度（提交人）、审批通过/驳回（当前审批人） |
| 已通过 | 所有审批层级均通过，库存已调整 | 查看、撤销 |
| 已驳回 | 某一级审批人驳回 | 修改（保存后恢复原审批流程）、删除 |
| 已撤销 | 已通过后被撤销，库存已恢复 | 编辑（重新提交）、删除 |

### 与现有功能的关系

- 原「已提交」状态拆分为「待审批」→「审批中」→「已通过」多个阶段，支持多级审批
- 库存调整时机从「提交时」变更为「所有审批层级通过时」
- 撤销功能仅对「已通过」状态生效，逻辑不变
- 驳回后提交人修改保存即恢复原审批流程（回到驳回前的审批节点继续），无需重新发起审批
- 审批流程配置（审批人、审批层级等）由独立审批模块管理，本功能仅负责发起审批和接收审批结果

### Story 1: 提交盘点单发起审批

**用户故事**
```
作为 仓管员
我想要 提交盘点单后自动进入审批流程，而不是直接调整库存
以便 盘点结果经过审批人确认后才生效，减少因录入错误导致的库存频繁波动
```

**优先级**：P0
**故事点估算**：3

**验收标准**
- [ ] AC1: 盘点单提交时，执行现有的库存校验逻辑（盘亏项当前库存 ≥ 减少数量），校验通过后盘点单状态变为「待审批」
- [ ] AC2: 提交后不执行库存调整，库存调整推迟到所有审批层级通过后执行
- [ ] AC3: 提交成功后向审批模块发起审批请求，携带盘点单ID、提交人、盘点摘要信息
- [ ] AC4: 提交成功后页面提示"已提交审批，请等待审批结果"
- [ ] AC5: 审批模块开始流转后（第一级审批人开始处理），盘点单状态自动变为「审批中」

**技术说明**
- 对接审批模块的发起审批接口，审批业务类型标识为「物料盘点」
- 提交时的库存校验逻辑保持不变，仅将库存调整执行时机后移
- 「待审批」→「审批中」的状态变更由审批模块回调通知

---

### Story 2: 审批通过后执行库存调整

**用户故事**
```
作为 审批人
我想要 所有审批层级通过后，系统自动执行库存调整
以便 确认盘点数据无误后库存才发生变化，保障数据准确性
```

**优先级**：P0
**故事点估算**：5

**验收标准**
- [ ] AC1: 多级审批场景下，每一级审批通过后盘点单保持「审批中」状态，流转至下一级审批人
- [ ] AC2: 所有审批层级均通过后，盘点单状态从「审批中」变为「已通过」
- [ ] AC3: 状态变更后自动执行库存调整（与原提交逻辑一致）：更新当前库存和可用库存
- [ ] AC4: 盘亏项按FIFO关联入库单扣减，盘盈项自动生成入库单（逻辑与原提交时一致）
- [ ] AC5: 执行库存调整前再次校验当前库存充足性，若不满足则标记为「审批异常」并通知提交人
- [ ] AC6: 库存调整在事务中完成，全部成功或全部回滚

**技术说明**
- 通过审批模块的回调接口接收审批结果，区分「单级通过（继续流转）」和「全部通过（终审通过）」
- 由于提交到最终审批通过之间可能存在较长时间差（多级审批），需在执行库存调整前重新校验库存
- 审批异常状态需通知提交人重新处理（撤销后重新盘点）

### Story 3: 审批驳回盘点单

**用户故事**
```
作为 审批人
我想要 驳回存在问题的盘点单并填写驳回原因
以便 仓管员了解问题所在并修正后继续审批流程
```

**优先级**：P0
**故事点估算**：3

**验收标准**
- [ ] AC1: 任意一级审批人驳回时，盘点单状态从「审批中」变为「已驳回」
- [ ] AC2: 驳回原因保存在盘点单记录中，在盘点单详情页可查看
- [ ] AC3: 已驳回的盘点单，提交人可直接修改盘点数据，编辑页面加载原有数据，系统库存刷新为当前最新值
- [ ] AC4: 修改保存后盘点单状态恢复为「审批中」，回到驳回前的审批节点继续审批流程（非重新发起）
- [ ] AC5: 已驳回的盘点单可删除（放弃本次盘点）

**技术说明**
- 驳回原因由审批模块传入，本功能负责存储和展示
- 修改保存后调用审批模块的「恢复审批」接口，将审批流程回退到驳回节点继续流转
- 需记录驳回前的审批节点信息（由审批模块维护），以便恢复时定位到正确的审批层级

### Story 4: 取消审批

**用户故事**
```
作为 仓管员
我想要 在审批流程开始流转之前取消已提交的审批
以便 发现盘点数据有误时能及时撤回，不需要等审批人驳回
```

**优先级**：P0
**故事点估算**：3

**验收标准**
- [ ] AC1: 「待审批」状态的盘点单，提交人可执行「取消审批」操作
- [ ] AC2: 取消审批前弹出确认弹窗，提示"取消后盘点单将回到草稿状态，可重新编辑后提交"
- [ ] AC3: 确认取消后，调用审批模块的撤销审批接口，取消进行中的审批流程
- [ ] AC4: 取消成功后盘点单状态变为「草稿」，可重新编辑和提交
- [ ] AC5: 若审批已开始流转（状态为「审批中」），取消审批失败，提示"审批已在处理中，无法取消"

**技术说明**
- 对接审批模块的撤销/取消审批接口
- 取消审批是否允许取决于审批模块返回的审批流程状态

### Story 5: 盘点单列表及详情适配审批状态

**用户故事**
```
作为 仓管员/车队管理员
我想要 在盘点单列表中看到审批相关的状态和信息
以便 了解每张盘点单的审批进度和结果
```

**优先级**：P1
**故事点估算**：3

**验收标准**
- [ ] AC1: 盘点单列表状态筛选项增加「待审批」「审批中」「已通过」「已驳回」，移除原「已提交」
- [ ] AC2: 列表中状态列展示新状态标签，各状态使用不同颜色区分（待审批-橙色、审批中-蓝色、已通过-绿色、已驳回-红色）
- [ ] AC3: 盘点单详情页展示审批信息区域：审批状态、当前审批节点/审批人、审批时间、驳回原因（如有）
- [ ] AC4: 详情页操作按钮根据状态动态显示：待审批显示「取消审批」、审批中无操作按钮、已通过显示「撤销」、已驳回显示「修改」「删除」
- [ ] AC5: 列表支持按新状态筛选

**UI/UX 说明**
- 状态标签颜色：草稿-灰色、待审批-橙色、审批中-蓝色、已通过-绿色、已驳回-红色、已撤销-灰色
- 详情页新增「审批信息」卡片，位于盘点基本信息下方
- 审批中状态可展示审批进度（已通过N级/共M级）

**用户故事**
```
作为 车队管理员
我想要 查看盘点单的审批操作记录
以便 追溯审批过程，了解谁在什么时间做了什么审批决定
```

**优先级**：P2
**故事点估算**：2

**验收标准**
- [ ] AC1: 盘点单详情页的操作日志中记录审批相关事件：提交审批、取消审批、审批通过、审批驳回
- [ ] AC2: 日志包含：操作类型、操作人、操作时间、备注（如驳回原因）
- [ ] AC3: 日志按时间倒序排列

## 状态操作权限矩阵

| 操作 | 执行角色 | 前置状态 | 结果状态 |
|------|---------|---------|---------|
| 提交 | 仓管员（提交人） | 草稿 / 已撤销 | 待审批 |
| 取消审批 | 仓管员（提交人） | 待审批 | 草稿 |
| 审批流转 | 审批模块自动 | 待审批 | 审批中 |
| 单级审批通过 | 当前级审批人 | 审批中 | 审批中（流转下一级） |
| 终审通过 | 最后一级审批人 | 审批中 | 已通过 |
| 审批驳回 | 当前级审批人 | 审批中 | 已驳回 |
| 修改保存（驳回后） | 仓管员（提交人） | 已驳回 | 审批中（恢复原审批节点） |
| 撤销 | 仓管员（提交人） | 已通过 | 已撤销 |
| 编辑 | 仓管员（提交人） | 草稿 / 已撤销 | 草稿 |
| 删除 | 仓管员（提交人） | 草稿 / 已驳回 / 已撤销 | - |

## 与现有Story的影响说明

| 原Story | 影响 |
|---------|------|
| Story 4: 提交盘点单并校验库存 | 提交不再直接执行库存调整，改为发起审批；库存调整移至所有审批层级通过后执行 |
| Story 5: 查看盘点单列表 | 状态筛选项变更，列表增加审批相关信息列，新增「审批中」状态 |
| Story 8: 撤销已提交的盘点单 | 撤销仅对「已通过」状态生效，「待审批」状态使用「取消审批」替代 |
| Story 9: 重新编辑已撤销的盘点单 | 「已驳回」状态修改保存后恢复原审批流程（非重新发起），「已撤销」状态重新提交则发起新审批 |

## 与其他模块的协作

- **审批模块**：本功能对接审批模块的以下接口：
  - 发起审批（提交时调用）
  - 撤销/取消审批（取消审批时调用）
  - 恢复审批（驳回后修改保存时调用，回到驳回前的审批节点继续）
  - 审批结果回调（接收审批流转、单级通过、终审通过、驳回等通知）
- 审批流程的配置（审批人、审批层级、审批条件等）由审批模块独立管理，不在本功能范围内
