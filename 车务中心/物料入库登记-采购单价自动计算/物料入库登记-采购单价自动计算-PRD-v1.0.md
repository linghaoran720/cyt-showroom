# 物料入库登记-采购单价自动计算

## 一、产品概述

### 1.1 产品定义
在 **车务中心 - 物料入库** 的登记与修改流程中，为物料明细统一 **入库数量、采购单价、采购总价（以下称「金额」）** 三者的互算与联动：**两有值自动算第三项**；**三者均有值** 时改单价或改数量实时更新金额，改金额则不改动单价与数量；**三者均有值后若清空任意一项，再修改另外两个字段中任一项或两项，则按「两有一空」补算被清空项**（含清空金额后改价/改量 → 补金额），保证明细闭环、减少单价漏填对 **出库** 等环节的影响。

### 1.2 产品目标
- 消除客户手工换算采购单价的操作成本。
- 避免「总价有值、单价为空」进入库存链路，降低出库单价必填时的业务阻塞与口径争议。

### 1.3 核心价值

| 维度 | 价值 |
|------|------|
| 效率 | 支持按「数量 + 总价」习惯录入，系统自动回填单价 |
| 数据质量 | 明细数量、单价、总价保持一致，利后续出库与对账 |
| 体验 | 三满态下改金额不「猜」改单价还是数量，避免误改；两有一空仍自动互算 |

## 二、用户分析

### 2.1 主要用户角色

| 角色 | 职责 | 使用场景 |
|------|------|---------|
| 仓库/车务登记人员 | 录入入库明细 | 按供应商/采购习惯填写数量、总价或单价 |
| 出库操作人员 | 办理物料出库 | 依赖入库侧成本或单价类信息完整性 |

### 2.2 用户痛点
1. 先填 **数量** 与 **总价** 时，系统不会反算 **单价**，需自行计算。
2. **单价** 留空保存后，后续 **出库** 等处 **单价必填** 时缺乏可靠依据，业务难以补救。

---

## 三、核心功能

### 3.1 功能清单

| 功能 | 描述 |
|------|------|
| 两有算一无 | 单价、数量、金额 **任两格合法有值** 时，自动补算 **空缺** 项（金额 = 单价 × 数量） |
| 三满改价/量 → 动金额 | 三格均有值时，用户改 **单价** 或 **数量**，**金额** 按 **单价 × 数量** **实时**更新 |
| 三满仅改金额 → 价量不动 | 三格均有值时，用户只改 **金额**，**单价、数量不变** |
| 三满清空一项后再改「另两项」 | 三满后用户 **清空单价/数量/金额中任意一项**，再修改 **未清空的两项** 中 **任一或两个**，**补算** 被清空项（与同表「两有一空」） |
| 边界与保存一致 | **正数**校验（见 §5.2）；**单价/金额 2 位、数量 3 位四舍五入**；除零、非法值不计算；保存与二次打开一致 |

## 四、用户场景

### 场景1：按总价录入一行物料

**用户**: 仓库登记人员  
**目标**: 快速完成一行入库明细且单价正确落库  

**流程**:
1. 打开 **入库登记**，选择物料，填写 **入库数量**（如 22）。
2. 填写 **采购总价**（如 333）。
3. 系统 **自动** 将 **采购单价** 填为 **333 ÷ 22**，并按 **§5.2** **单价 2 位小数、四舍五入** 展示（示例：15.14）。
4. 点击 **确定** 保存。

**预期结果**: 保存后该行 **数量、单价、金额** 自洽；再次打开仍正确；后续出库不被空单价卡住（在现有业务规则下）。

---

### 场景2：三字段都填好后改单价或数量

**用户**: 仓库登记人员  
**目标**: 微调单价或数量时，金额跟着变  

**流程**:
1. 明细行 **单价、数量、金额** 均已填写。
2. 用户修改 **单价** 或 **数量**。
3. **金额** 实时变为 **单价 × 数量**。

**预期结果**: 单价、数量以用户输入为准，金额按 **二者乘积** 计算并以 **§5.2** **金额 2 位、四舍五入** 展示。

---

### 场景3：三字段都填好后只改金额

**用户**: 仓库登记人员  
**目标**: 只想改总价，不希望系统在「单价 / 数量」里自动替我做二选一  

**流程**:
1. 明细行三字段均有值。
2. 用户只修改 **金额**，不改单价与数量。

**预期结果**: **单价、数量保持修改金额前的值不变**；可能出现 **单价 × 数量 ≠ 金额** 的暂时不一致，以用户明确以 **金额** 为当前录入结果（若业务后续要求保存前强校验，可单列规则；本版默认允许该状态直至用户再通过改价/改量或清空再录来对齐）。

---

### 场景4：三满后清空任意一项，再改「另外两个」

**用户**: 仓库登记人员  
**目标**: 先清掉一项错的，通过改 **剩下两项** 把空项算回来  

**流程**:
1. 三字段曾均有值；用户 **清空其中任意一项**（单价、数量或金额）。
2. 用户修改 **另外两个字段** 中的 **任意一个或两个**（例如：清空单价后，可改数量或改金额，或两个都改）。

**预期结果**: 与 **两有一空** 一致——缺 **单价** → **单价 = 金额 ÷ 数量**；缺 **数量** → **数量 = 金额 ÷ 单价**；缺 **金额** → **金额 = 单价 × 数量**；结果均按 **§5.2** 做 **正数** 与 **小数位四舍五入**；除零不计算。

## 五、功能详细说明

### 5.1 换算规则总表（核心）

明细行三个字段：**采购单价**（下称单价）、**入库数量**（下称数量）、**采购总价**（下称金额）。恒等关系：**金额 = 单价 × 数量**（用于「两有一空」时的补算；**数值与精度见 §5.2**）。

| 当前状态 | 用户操作 | 系统行为 |
|----------|----------|----------|
| **两有一空** | 编辑任一带值字段，维持「两有一空」 | **自动补算空缺**：缺金额 → 金额 = 单价 × 数量；缺单价 → 单价 = 金额 ÷ 数量；缺数量 → 数量 = 金额 ÷ 单价 |
| **三均有值** | 改 **单价** 或 **数量** | **金额** 实时更新为 **单价 × 数量** |
| **三均有值** | 仅改 **金额** | **单价、数量均不变** |
| **三均有值** → 用户 **清空任意一项** | 再改 **另外两个字段** 中 **任一或两项** | 行内已处于 **两有一空**，按上表首行 **自动补算被清空项**（不限于「只改金额」） |

**实现提示**：三满态下须区分「用户正在改的是金额还是价/量」，避免 **三满** 下改金额时误触发「用金额÷数量重算单价」。用户 **清空一项** 后状态退化为 **两有一空**，此后对 **已填的两项** 任意编辑均应重算 **空缺项**（与用户修改顺序无关：先试算能否唯一确定第三项）。

**校验规则**:
- 除数为 0、空值、非正数导致无法互算时：**不计算**、不抛未捕获异常；提示可与现网一致。
- **§5.2** 未满足时不得写入非法结果。

### 5.2 数值、精度与校验

以下适用于 **互算结果回填** 及 **展示/保存** 口径（前后端一致；内部可先高精度运算再对落库字段做舍入）。

| 项目 | 要求 |
|------|------|
| **有效值范围** | **数量、单价、金额** 参与计算或保存时均为 **正数**（**> 0**）。0、负数、非数字：按现网方式 **提示/拦截**，**不参与**互算（作除数时等同非法）。 |
| **单价** | **2 位小数**；任一环节定标时 **四舍五入** 到 2 位。 |
| **金额（采购总价）** | **2 位小数**；任一环节定标时 **四舍五入** 到 2 位。 |
| **数量** | **3 位小数**；任一环节定标时 **四舍五入** 到 3 位。 |

**说明**：用户手工输入可先按控件规则校验，再与互算结果统一走上述精度；若计算链路为多步，以对 **最终写入单元格/提交字段** 的舍入为准，避免与后端存储不一致。

---

### 5.3 与知识库原文的差异说明

知识库当前表述侧重 **金额 = 数量 × 单价** 单向。**本 PRD** 采用 **§5.1** 表 **两有一空互算** 与 **三满改金额不动价量**，以及 **§5.2 数值与精度**；请在发版后 **同步回写知识库** 中「登记入库流程」对应小节。

---

### 5.4 保存与后端一致性（建议）

**功能描述**: 保存请求中，对「数量、总价均有值」的明细，若 **单价** 仍为空，服务端可按同一公式补算或返回明确错误；**推荐** 以前端已填充 + 后端兜底校验 **双保险**，避免并发或绕过前端。

**影响页面**: 保存接口逻辑  
**影响角色**: 研发/测试  

**校验规则**:
- 与现网「必填字段」定义对齐：若 **单价** 在接口层仍为必填，则 **禁止** 在数量、总价合法时出现空单价提交。
- **单价 / 金额 / 数量** 的 **正数** 与 **小数位、四舍五入** 规则与 **§5.2** 一致，前后端同一套口径。

## 六、验收标准（汇总）
- [ ] **两有一空**：三种空缺组合下均能自动补算第三项；除零/非法/非正数不崩溃。
- [ ] **三满**：改 **单价** 或 **数量**，**金额** 实时 = **单价 × 数量**（结果按 **§5.2** 舍入）。
- [ ] **三满**：仅改 **金额**，**单价、数量** 不变。
- [ ] **三满** 后清空 **单价 / 数量 / 金额** 中 **任意一项**，再改 **另外两个** 中 **任一或两项**，空项按 **两有一空** 补回，且符合 **§5.2**。
- [ ] **正数**：数量、单价、金额不允许 0/负数落库（与现网校验一致）。
- [ ] **精度**：单价与金额为 **2 位小数**，数量为 **3 位小数**，均为 **四舍五入**；保存并二次打开与展示一致。
- [ ] 回归：出库等对单价完整性的依赖不弱于改动前。
