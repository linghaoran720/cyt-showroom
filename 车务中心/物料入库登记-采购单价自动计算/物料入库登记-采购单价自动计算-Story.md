# 物料入库登记-采购单价自动计算 - 用户故事

## 用户故事

### Story 1：两有值自动算第三项

**作为** 仓库/车务登记人员  
**我想要** 在明细行里 **单价、数量、金额（采购总价）任意两项有合法值** 时，系统自动算出 **空缺那一项**  
**以便于** 无需手算、也少出现漏填  

**验收条件**：
- [ ] Given **数量**、**金额** 有值、**单价** 为空，When 用户修改数量或金额（与线上一致的触发时机），Then **单价 = 金额 ÷ 数量**（小数位与现网一致）
- [ ] Given **单价**、**金额** 有值、**数量** 为空，When 用户修改单价或金额，Then **数量 = 金额 ÷ 单价**（数量小数位与现网一致）
- [ ] Given **单价**、**数量** 有值、**金额** 为空，When 用户修改单价或数量，Then **金额 = 单价 × 数量**
- [ ] Given 作为除数的 **数量或单价为 0 / 空 / 非法**，When 需要除法，Then **不计算、不崩溃**（可轻量提示）

**优先级**：P1

---

### Story 2：三均有值时的改价与改量

**作为** 仓库/车务登记人员  
**我想要** 在 **三个字段都已填写** 时，改 **单价** 或 **数量** 能立刻看到 **金额** 跟着变；只改 **金额** 时不要让系统在单价和数量里「猜」改谁  
**以便于** 行为可预期、不误改单价或数量  

**验收条件**：
- [ ] Given **单价、数量、金额均有值**，When 用户修改 **单价** 或 **数量**，Then **金额** **实时**更新为 **单价 × 数量**
- [ ] Given **单价、数量、金额均有值**，When 用户仅修改 **金额**，Then **单价**、**数量** **均不变**

**优先级**：P1

---

### Story 3：三均有值后清空一项，再改「另外两个」

**作为** 仓库/车务登记人员  
**我想要** 在 **三字段曾都填过** 后，若 **清空其中任意一项**，再修改 **剩下两项里的任意一个或两个** 时，系统能 **补回被清空的那一项**  
**以便于** 修正录入错误时仍能闭环（不限于「只改金额」才算）  

**验收条件**：
- [ ] Given 曾三均有值，用户 **清空单价**，When 用户修改 **数量** 或 **金额**（或先后都改），Then **单价** 按 **两有一空** 规则补算（**单价 = 金额 ÷ 数量**，合法时）
- [ ] Given 曾三均有值，用户 **清空数量**，When 用户修改 **单价** 或 **金额**，Then **数量** 按 **数量 = 金额 ÷ 单价** 补算（合法时）
- [ ] Given 曾三均有值，用户 **清空金额**，When 用户修改 **单价** 或 **数量**，Then **金额** 按 **金额 = 单价 × 数量** 补算
- [ ] Given 用户完成登记并保存，When 再次打开该明细，Then 三字段与上述规则及存库一致

**优先级**：P1

## 边界说明

### 本次包含
- 入库 **登记**、**修改** 弹窗内物料明细行的 **数量、采购单价、采购总价（金额）** 联动规则：**两有算一无、三满改价/量动金额、三满改金额不动价量、三满后清空任意一项再改「另外两个」则补算清空项**
- 除零、非正数、空值的防御与轻量提示（文案在 PRD）

### 本次不包含
- 历史数据批量清洗脚本（可另立项）
- 出库页自身校验规则改造（若出库强依赖本项目落库后的单价，则以本项填充单价间接解决）
- 导入模板逻辑（若导入通道也存在同类字段，可单列 follow-up）

## 影响范围

- **影响页面**：车务中心 - 物料入库 - 登记/修改
- **影响角色**：有物料入库新增、修改权限的用户
- **数据变更**：不改变表结构；明细行写入的 **采购单价** 由空变为有值（在客户按总价录入场景下）
