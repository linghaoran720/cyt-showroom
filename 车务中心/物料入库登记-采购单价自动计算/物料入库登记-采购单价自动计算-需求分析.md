# 物料入库登记-采购单价自动计算 - 需求分析

## 1. 工单信息
- **工单来源**：客户工单
- **问题描述**：物料入库登记时，系统不能根据「入库数量」和「采购总价」反算「采购单价」。客户需自行换算；若单价留空，后续出库环节单价必填时无法还原原始单价。
- **优先级**：P1（高）— 影响数据完整性与 downstream 出库闭环

## 2. 现有功能逻辑

### 2.1 当前实现
- 车务中心 **物料入库** 支持登记入库：明细中包含 **入库数量**、**采购单价**、**采购总价** 等字段（知识库字段：`in_nums`、`buy_unit_prices`、总价相关字段）。
- 知识库描述的登记交互为：**采购总价由「入库数量 × 采购单价」系统自动计算**。
- 实际使用中客户存在 **先掌握总价与数量、再需得到单价** 的习惯路径；当前若未填单价而仅填总价，**系统未按「总价 ÷ 数量」自动回填单价**，导致：
  - 登记阶段体验差；
  - **单价缺失进入库存/出库链路**，出库单价必填时业务无法可靠溯源。

### 2.2 相关页面
- **物料入库-登记/修改弹窗**：物料明细表格（入库数量、采购单价、采购总价等）
- **物料出库**（关联影响）：依赖入库侧写入的单价或成本口径数据时，单价为空会引发填报受阻或口径不一致

### 2.3 数据流
`登记/修改入库明细` → 持久化至 `MaterialInboundBill` / 明细及 `MaterialStock` → 出库等后续环节读取或校验单价类字段

## 3. 客户痛点分析

### 3.1 问题场景
- 采购按 **整单总价** 或习惯先录 **数量 + 总价**，期望系统自动算出 **单价**。
- 用户未手算或未填单价即保存，后续 **出库要求单价必填** 时，**缺少可信赖的单价来源**。

### 3.2 根本原因
- 产品与实现侧重 **单价 × 数量 → 总价** 单向联动，**缺少「数量 + 总价 → 单价」** 的反算与落库保障。
- 字段间联动规则未覆盖客户真实录入顺序，导致 **单价可空但总价有值** 的不一致状态。

### 3.3 影响范围
- **影响用户**：仓库/车务登记人员、后续出库操作人员、财务对账相关人员
- **影响频率**：中高（凡按总价习惯录入即触发）
- **严重程度**：高（数据链断裂、出库阻塞或事后估算错误）

## 4. 解决方案

### 4.1 方案说明（最小改动）
在 **入库登记 / 修改** 的物料明细行内，统一 **入库数量、采购单价、采购总价（金额）** 三者的互算与联动规则（见下），并在保存时避免出现「金额或单价缺失且无法推导」的无效组合（在业务允许前提下）。

### 4.2 单价 / 数量 / 金额换算规则（产品定稿）

以下「金额」即明细 **采购总价**。

| 状态 | 用户操作 | 系统行为 |
|------|----------|----------|
| **仅两格有值**（一空） | 任一已填项变更使仍保持「两有一空」 | 用 **金额 = 单价 × 数量** 关系 **自动补算空缺项**：缺金额则 **金额 = 单价 × 数量**；缺单价则 **单价 = 金额 ÷ 数量**；缺数量则 **数量 = 金额 ÷ 单价**（精度与现网字段小数位一致，除零不计算） |
| **三格均有值** | 修改 **单价** 或 **数量**（实时） | **金额** 按 **单价 × 数量** **实时重算**；**单价与数量** 以用户输入为准 |
| **三格均有值** | 修改 **金额** | **单价、数量均不变**（无法唯一确定应改单价还是数量，避免误改） |
| **曾三格均有值** → 用户 **清空任意一项**（单价 / 数量 / 金额 均可） | 再修改 **另外两个字段**（未清空的两项）**中的任意一个或两个** | 当前行为等价于 **两有一空**：按 **4.2** 首行公式 **补算被清空的那一项**（缺单价 → 金额÷数量；缺数量 → 金额÷单价；缺金额 → 单价×数量） |

**规则小结**：两有算一无；三满改单价或改数量只驱动金额；三满改金额不动单价数量；**三满后清空任意一项，再改「另外两个」中任一项或两项，均触达对清空项的补算**（与首行「两有一空」一致，本条强调从三满退化的路径）。

### 4.3 改动内容（实现项）
- [ ] 明细行按 **4.2** 表实现前端联动（触发时机与现网输入控件一致，如变更/失焦，避免死循环）。
- [ ] **采购单价** 小数位、**金额** 小数位、**数量** 小数位与现网一致；除零、非正数、非法输入不崩溃，必要时节流提示（见 PRD）。
- [ ] 保存时：对可推导仍为空的必填项，以前端预填 + 后端与同公式兜底校验为推荐。

### 4.4 影响评估
- **影响页面**：物料入库登记/修改弹窗（主）；列表展示一般仅展示结果，可不做改动
- **影响角色**：有 `materialInStockAdd` / `materialInStockUpdate` 权限的用户
- **数据迁移**：一般 **不需要**；可选：批处理修复历史「总价有值、单价为空」的明细（非本工单必选，可单列技术债）

## 5. 验收标准
- [ ] **两有一空**：任意两字段合法有值时，第三字段按 **4.2** 自动补算（缺单价 / 缺数量 / 缺金额 三种均能覆盖）。
- [ ] **三满**：改 **单价** 或 **数量**，**金额** 实时变为 **单价 × 数量**。
- [ ] **三满**：仅改 **金额**，**单价、数量** 保持不变。
- [ ] **三满** 后 **清空任意一项**，再修改 **另外两个字段**（单行内价/量/额）**任一或组合修改**，被清空项按 **4.2** 正确补回（含：清空金额后改单价或数量 → 补金额；清空单价后改数量或金额 → 补单价；清空数量后改单价或金额 → 补数量）。
- [ ] 无除零、非法输入崩溃；保存后与二次打开数据与规则一致。
- [ ] 回归：**出库** 等对单价完整性的依赖场景不弱于改动前。

## 6. 风险评估
| 风险项 | 影响程度 | 应对策略 |
|--------|----------|----------|
| 三满改金额不联动，用户误以为未生效 | 低 | 可在金额失焦时轻提示「已按您的金额保存，未改动单价与数量」或帮助文案（PRD 可选） |
| 规则与旧版「改总价必重算单价」习惯不一致 | 中 | 以 **4.2 三满改金额不动** 为准，培训/发版说明；两有一空仍照常互算 |
| 舍入导致「单价×数量≠总价」差 0.01 | 低 | 与现有金额舍入策略统一；必要时以总价为基准展示 |
| 历史数据单价仍为空 | 中 | 运营/技术方案：可选补数脚本或出库时允许按库存批次规则兜底（超出本工单可记入后续） |
