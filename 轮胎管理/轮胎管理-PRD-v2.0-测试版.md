## 一、产品概述

### 1.1 产品定义

轮胎管理模块是财运通 TMS 系统中用于管理车辆轮胎全生命周期的功能模块。支持从轮胎采购入库、领用装车、回收、报废到售卖的完整流程，并与财务系统集成，实现报销单和收据的自动创建。

MVP 阶段轮胎管理模块独立运作，与车辆管理模块轻量集成（仅查询），维修单据采用手动关联方式。

### 1.2 产品目标

- 建立轮胎资产全生命周期管理能力，替代现有人工台账
- 实现轮胎与车辆的精准绑定，支持安装位置追踪
- 打通采购报销和售卖收据的财务流程
- 为管理者提供轮胎库存和成本分析数据

### 1.3 核心价值

| 维度 | 价值 |
|------|------|
| 资产管理 | 实时掌握每条轮胎的状态、位置和成本，消除人工台账误差 |
| 财务管理 | 自动生成报销单和收据，减少财务对账工作量 |
| 运营管理 | 提供库存统计和成本分析，支持管理决策 |
| 追溯能力 | 完整的事件日志，每个操作可回溯到具体操作人和时间 |

## 二、用户分析

### 2.1 主要用户角色

| 角色 | 职责 | 主要操作 |
|------|------|---------|
| 仓库管理员 | 负责轮胎入库、出库管理 | 轮胎入库、修改入库单、创建报销单 |
| 车队调度员 | 负责轮胎领用和回收 | 轮胎领用、轮胎回收、维修记录 |
| 车队管理员 | 负责轮胎报废和售卖决策 | 轮胎报废、轮胎售卖、查看统计 |
| 财务人员 | 负责财务单据审批 | 审批收据、查看报销状态 |
| 系统管理员 | 负责权限配置 | 权限管理 |

### 2.2 用户痛点

1. **台账管理混乱**：现有轮胎信息靠 Excel 或手工记录，库存不准，找不到轮胎
2. **状态不透明**：不清楚某条轮胎装在哪辆车的哪个位置，出问题无法快速定位
3. **财务流程割裂**：报销和售卖收款靠人工处理，容易漏单错单
4. **成本不可知**：无法统计某辆车、某个时段的轮胎总成本

## 四、轮胎状态机

### 4.1 状态定义

| 状态值 | 状态名 | 说明 |
|--------|--------|------|
| 1 | 入库 | 轮胎已入库，在库存中 |
| 2 | 已领用 | 轮胎已装到车上 |
| 3 | 已回收 | 轮胎从车辆回收到库存 |
| 4 | 已作废 | 轮胎已报废 |
| 5 | 售卖待确认 | 已发起售卖，等待收据审批 |
| 6 | 已卖出 | 收据审批通过，轮胎已完成售卖 |

### 4.2 状态流转

```
入库（1）→ 已领用（2）→ 已回收（3）→ 已作废（4）→ 售卖待确认（5）→ 已卖出（6）
  │                          │               │                  ▲
  └──────────────────────────┴───────────────┘
           可直接售卖（需先回收，已领用不可直接卖）
```

| 当前状态 | 允许的操作 | 不允许的操作 |
|---------|------------|------------|
| 入库 | 领用、卖出 | 作废、回收 |
| 已领用 | 回收 | 领用、卖出、作废 |
| 已回收 | 领用、卖出、作废 | — |
| 已作废 | 卖出 | 领用、回收 |
| 售卖待确认 | 无（等待收据审批） | 所有操作 |
| 已卖出 | 无（终态） | 所有操作 |

## 五、功能详细说明

### 5.1 轮胎入库

**功能描述**：用户在轮胎管理模块手动录入轮胎，支持单个录入和批量编号解析。

#### 5.1.1 入库单号生成规则

**规则**：网点名称 + 年后2位 + 月2位 + 日2位 + 流水号（4位）

**示例**：总部仓库260330001、北京分公司260330001

#### 5.1.2 入库抽屉弹窗 - 基本信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 入库网点 | 显示 | — | 当前用户所在网点，自动填充 |
| 使用范围 | 下拉 | 是 | 本级及下级（默认）/ 本级 / 全集团 |
| 入库来源 | 文本 | 是 | 如"采购"、"外部收购"等 |
| 联系人 | 文本 | 是 | 入库操作的联系人 |
| 入库时间 | 日期时间 | 是 | 默认当前时间 |
| 采购方 | 下拉 | 否 | 采购方信息 |
| 供应商 | 文本 | 否 | 供应商名称 |
| 供应商电话 | 文本 | 否 | 供应商联系电话 |
| 备注 | 文本域 | 否 | 入库备注 |

**布局**：
- 第一行：入库网点、使用范围、入库来源、联系人
- 第二行：供应商、供应商电话、入库时间、采购方
- 第三行：备注（文本域）

#### 5.1.3 入库抽屉弹窗 - 轮胎明细

**表格顶部批量填充区**（可一键应用到所有行）：品牌、尺寸、规格、型号、单价、轮胎类型

**明细表格列**：

| 序号 | 轮胎编号 | 品牌 | 尺寸 | 规格 | 型号 | 单价 | 轮胎类型 | 操作 |
|------|---------|------|------|------|------|------|---------|------|
| 1 | 用户录入 | 下拉 | 文本 | 文本 | 文本 | 数字 | 新胎/旧胎 | 删除 |

#### 5.1.4 批量录入编号

**触发**：点击"批量录入轮胎编号"按钮，打开对话框

**支持分隔符**：逗号、空格、换行符、制表符

**逻辑**：解析编号 → 去重去空 → 自动生成对应行数 → 填充编号字段

#### 5.1.5 校验规则

**前端**：
- 轮胎编号、品牌、尺寸、轮胎类型、单价、使用范围必填
- 同一入库单内编号不重复

**后端**：
- 轮胎编号全局唯一
- 入库单号生成防重

### 5.2 修改入库单

**入口**：轮胎列表 → 勾选任意一条轮胎 → 点击"修改入库单"

**逻辑**：系统读取所选轮胎的入库单号，加载该入库单下所有轮胎进入修改弹窗。

#### 5.2.1 修改权限规则

| 场景 | 可修改字段 |
|------|------------|
| 轮胎状态为"入库" | 全部字段 |
| 已报销（reimbursement_status=3） | 不可修改任何字段 |
| 售卖待确认 | 不可修改任何字段 |
| 未报销/部分报销 且 单价为空或0 | 仅可修改单价（后结算场景） |
| 未报销/部分报销 且 单价不为空 | 不可修改任何字段 |

**始终不可修改**：轮胎编号（唯一标识）、入库单号、当前状态

**UI 规则**：不可修改字段置灰显示，禁用点击和键盘输入，光标显示 `not-allowed`

---

### 5.3 轮胎领用

**功能描述**：将轮胎分配给车辆，指定安装位置。支持同时回收旧轮胎。

#### 5.3.1 基本信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 车牌号 | 下拉 | 是 | 选择车辆 |
| 司机 | 文本 | 否 | 自动带入车辆关联司机 |
| 领用人 | 文本 | 是 | 默认等于司机 |
| 领用时间 | 日期时间 | 是 | 默认当前时间 |
| 领用原因 | 文本域 | 是 | — |
| 码表里程 | 数字 | 是 | 领用时的车辆码表读数 |

#### 5.3.2 选择轮胎

- 输入入库单号或轮胎编号搜索，支持回车触发
- 搜索范围：状态为"入库"或"已回收"，且使用范围包含当前网点的轮胎
- 安装位置必填，可选：前左、前右、后左1、后右1、后左2、后右2、备胎

#### 5.3.3 当前轮胎（同步回收）

- 系统自动展示该车辆已领用的轮胎列表
- 用户可勾选"回收"或"回收并作废"
- 勾选后，该位置不再视为冲突

#### 5.3.4 校验规则

| 类型 | 规则 |
|------|------|
| 前端 | 轮胎不重复、位置不冲突（未勾选回收时标红提醒）、必填字段非空 |
| 后端 | 轮胎状态校验、使用范围校验、位置冲突校验 |

---

### 5.4 轮胎回收

**功能描述**：将已领用的轮胎从车辆回收。支持批量操作，回收完成后可快速发放新轮胎。

#### 5.4.1 操作流程

1. 列表勾选轮胎 → 点击"轮胎回收"
2. 弹窗：选择要回收的轮胎（仅显示已领用状态）
3. 点击"下一步" → 进入批量执行弹窗
4. 实时显示执行进度（待处理 / 成功 / 失败）
5. 全部完成后，弹出"领用轮胎"按钮（可选的快速发放入口）

#### 5.4.2 回收后快速发放

**触发**：回收完成弹窗中点击"领用轮胎"

**自动带入**：车牌号、司机、领用人、领用原因（显示"轮胎替换"）

**需填写**：码表里程

**轮胎选择表格**：以回收轮胎为行，每行可搜索并选择新轮胎（包含已回收状态的轮胎）

#### 5.4.3 批量操作限制

- 默认上限 200 条
- 状态非"已领用"的轮胎：显示警告，提供"取消"和"继续"选项

---

### 5.5 轮胎报废

**功能描述**：将已回收的轮胎标记为报废，记录报废原因。

**前置条件**：轮胎状态为"已回收"

**报废原因**：磨损过度 / 损伤无法修复 / 老化 / 其他（选择"其他"需补充说明）

**批量逻辑**：
- 支持批量操作，默认 200 条上限
- 每条轮胎必须填写报废原因，否则"确认报废"按钮禁用
- 状态不符合的轮胎显示警告提示

---

### 5.6 轮胎售卖

**功能描述**：将轮胎售卖给外部客户，自动创建收据并将轮胎状态设为"售卖待确认"。

**可售卖状态**：入库、已回收、已作废（已领用不可直接售卖，需先回收）

#### 5.6.1 顶部信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 售卖对象 | 文本 | 是 | 客户/公司名称 |
| 联系人 | 文本 | 是 | — |
| 联系电话 | 文本 | 是 | — |

#### 5.6.2 售卖明细表格

| 字段 | 说明 |
|------|------|
| 轮胎编号、品牌、规格、采购价格 | 显示，不可编辑 |
| 售卖价格 | 必填，支持两位小数 |
| 价格差异原因 | 当售卖价格 < 采购价格时必填；否则可选 |

#### 5.6.3 收据与状态联动

| 事件 | 轮胎状态变化 |
|------|------------|
| 确认售卖 | → 售卖待确认（自动创建收据） |
| 收据审批通过 | → 已卖出 |
| 收据拒绝/取消 | → 恢复售卖前状态（入库/已回收/已作废） |

---

### 5.7 创建报销单

**功能描述**：根据轮胎入库单创建财务报销单据，支持跨入库单批量报销，按采购价格比例分摊报销金额。

#### 5.7.1 分摊规则

- **公式**：分摊金额 = 报销总金额 × (该轮胎采购价格 / 所有轮胎采购价格之和)
- **精度**：4舍5入，保留两位小数
- **余数处理**：余数分配给分摊金额最大的轮胎

**示例**：报销 30 元，A 轮胎 100 元，B 轮胎 50 元 → A 分摊 20.00 元，B 分摊 10.00 元

#### 5.7.2 对话框结构

**顶部**：报销总金额（必填）、报销说明（选填）

**明细表格**：

| 字段 | 说明 |
|------|------|
| 轮胎编号、入库单号、采购价格、采购数量、采购小计 | 显示，不可编辑 |
| 分摊报销金额 | 自动计算，实时更新 |
| 报销状态 | 显示当前报销状态 |

**底部统计**：轮胎总数、采购总金额、报销总金额、未报销金额

#### 5.7.3 校验规则

| 类型 | 规则 |
|------|------|
| 前端 | 报销总金额必填且 > 0，报销总金额 ≤ 采购总金额 |
| 后端 | 轮胎状态正常校验、报销金额校验、财务系统可用性校验 |

---

### 5.8 取消操作

**功能描述**：撤销已执行的操作，将轮胎状态恢复到操作前。

**入口**：轮胎列表 → 勾选轮胎 → 点击"取消操作" → 选择操作类型（下拉菜单）

**状态回退规则**：

| 取消操作 | 原状态 | 目标状态 |
|---------|-------|---------|
| 删除入库 | 入库 | 已删除（软删除） |
| 取消领用 | 已领用 | 入库 / 已回收（取决于领用前状态） |
| 取消回收 | 已回收 | 已领用（恢复原车辆关联） |
| 取消报废 | 已作废 | 已回收 |
| 取消售卖 | 售卖待确认 | 入库 / 已回收 / 已作废（取决于售卖前状态） |

---

### 5.9 维修记录

**功能描述**：手动关联维修单据，记录本次维修成本，累计更新轮胎的维修次数和总成本。

**适用状态**：任何状态均可记录维修

**关键规则**：
- 维修单据 ID 可选（不填时仅记录成本）
- 同一轮胎不可重复关联同一维修单据
- 同一维修单据可关联多条不同轮胎
- 不创建维修事件记录，仅更新 tire_card 的 `repair_count` 和 `repair_cost`

### 5.10 轮胎列表页面

| 已作废 | 售卖、详情 |
| 售卖待确认 | 详情 |
| 已卖出 | 详情 |

默认筛选项：入库单号（文本）、轮胎编号（文本）、车牌号（文本）、轮胎状态（下拉）、领用人（文本）、入库时间（日期范围，默认近1个月）

#### 5.10.6 页面级操作按钮

| 按钮 | 前置条件 |

## 六、权限控制

### 6.1 功能权限

| 权限组 | 包含权限 |
|--------|---------|
| 入库管理权限 | 轮胎入库、修改入库单、删除入库 |
| 领用管理权限 | 轮胎领用、取消领用 |
| 回收管理权限 | 轮胎回收、取消回收 |
| 报废管理权限 | 轮胎报废、取消报废 |
| 售卖管理权限 | 轮胎售卖、取消售卖 |
| 财务管理权限 | 创建报销单、收据审批 |
| 维修管理权限 | 维修记录 |
| 统计分析权限 | 统计分析查看 |

### 6.2 数据权限

**按网点隔离**：轮胎归属入库网点

**按使用范围隔离**：
- 本级及下级：当前网点及其下级网点可查看和使用
- 本级：仅入库网点可查看和使用
- 全集团：集团内所有网点可查看和使用

**操作权限限制**：
- 领用/回收：当前用户所在网点必须在轮胎使用范围内
- 编辑/删除：仅入库网点用户可操作
