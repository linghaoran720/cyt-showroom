# 轮胎管理模块 - 需求分析

> **文档版本**: v2.0
> **创建日期**: 2026-03-28
> **更新日期**: 2026-03-28
> **需求类型**: 新能力探索
> **影响范围**: 车务中心-轮胎管理模块（完全独立）

---

## 一、需求背景

### 1.1 业务现状

**当前痛点**：
1. 轮胎作为车辆重要资产，缺乏全生命周期管理
2. 轮胎采购、领用、维修、报废等环节数据分散，无法追溯
3. 无法统计轮胎使用成本，不知道哪个品牌更耐用
4. 无法分析车辆轮胎消耗情况，不知道哪辆车成本最高
5. 轮胎库存管理混乱，经常出现重复采购或缺货

**业务诉求**：
- 需要一个完全独立的轮胎全生命周期管理系统
- 能够追溯每条轮胎的完整生命周期（采购→领用→回收→报废→报销→售卖）
- 支持轮胎的循环使用（领用→回收→再领用）
- 能够输出简单的统计分析报表
- 短期不依赖物料模块和维修模块，完全自闭环

### 1.2 设计原则

**核心原则**：
1. **一胎一号** - 使用轮胎出厂编号，全生命周期可追溯
2. **完全独立** - 不依赖物料模块和维修模块，自闭环运行
3. **状态驱动** - 通过状态流转管理轮胎生命周期，状态只能往下流转
4. **可循环** - 支持领用↔回收无限循环
5. **轻量化** - 不增加现场人员负担，只记录关键节点
6. **可扩展** - 预留入库单号，未来可与物料模块打通

**不做的事情**：
- ❌ 不做胎压、胎温、花纹深度等精细化管理
- ❌ 不做安全预警、强制保养提醒
- ❌ 不做复杂的轴位管理、动平衡、四轮定位
- ❌ 不做翻新、修补次数限制
- ❌ 不做复杂审批流
- ❌ 不做司机/线路/里程精细分析

### 1.3 与现有系统的关系

**短期（第一阶段）**：
- 轮胎管理完全独立，不与物料模块、维修模块联动
- 轮胎的采购、领用、回收、报废、报销、售卖全部在轮胎管理模块完成
- 司机仅需在维修登记中登记维修费用（如"换胎30元"），不涉及轮胎管理
- 轮胎管理中可选关联维修单，记录成本

**中期（第二阶段）**：
- 物料入库后可选提示生成轮胎卡片
- 维修登记页面增加"轮胎维护"入口

**长期（第三阶段）**：
- 物料模块增加序列号管理，全面打通数据

## 二、核心概念

### 2.1 入库单

每次轮胎采购入库生成一个入库单，包含批次信息和轮胎明细，未来与物料模块打通。

#### 入库单-基本信息

| 字段名 | 说明 | 必填 | 默认值 | 示例 |
|-------|------|------|-------|------|
| 入库单号 | 系统自动生成，用户可改，校验唯一性 | 是 | 系统生成 | 2603280001 |
| 采购日期 | 采购入库日期 | 是 | 当前日期 | 2026-03-28 |
| 供应商 | 供应商名称 | 否 | - | 某某轮胎经销商 |
| 所属网点 | 数据归属网点 | 是 | 当前网点 | 浩瀚物流 |
| 使用网点 | 数据可见范围 | 是 | 全集团 | 全集团 |
| 备注 | 备注信息 | 否 | - | - |

#### 入库单-轮胎明细

| 字段名 | 说明 | 必填 | 示例 |
|-------|------|------|------|
| 轮胎编号 | 出厂编号，用户手动录入，支持扫码枪 | 是 | DOT3A2BXXX0123 |
| 品牌 | 轮胎品牌 | 是 | 米其林 |
| 规格 | 轮胎规格 | 是 | 295/80R22.5 |
| 型号 | 轮胎型号 | 是 | XDA2+ Energy |
| 物料单位 | 固定"条"，不可修改 | 是 | 条 |
| 采购价格 | 单条轮胎采购价格 | 是 | 1,280.00 |

**入库单号生成规则**：
- 格式：年后2位 + 月2位 + 日2位 + 4位流水号
- 示例：2603280001（2026年03月28日第0001号）
- 系统自动生成，用户可修改
- 校验唯一性，不允许重复

### 2.2 轮胎卡片（核心主数据）

每条轮胎对应一张轮胎卡片，一胎一号：

| 字段名 | 说明 | 必填 | 默认值 | 示例 |
|-------|------|------|-------|------|
| 轮胎编号 | 出厂编号，用户手动录入 | 是 | - | DOT3A2BXXX0123 |
| 入库单号 | 关联入库单 | 是 | 自动关联 | 2603280001 |
| 品牌 | 轮胎品牌 | 是 | - | 米其林 |
| 规格 | 轮胎规格 | 是 | - | 295/80R22.5 |
| 型号 | 轮胎型号 | 是 | - | XDA2+ Energy |
| 物料单位 | 固定"条"，不可修改 | 是 | 条 | 条 |
| 采购价格 | 单条轮胎采购价格 | 是 | - | 1,280.00 |
| 采购日期 | 采购入库日期 | 是 | 入库单日期 | 2026-03-28 |
| 供应商 | 供应商名称 | 否 | 入库单供应商 | 某某轮胎经销商 |
| 所属仓库 | 当前所在仓库 | 是 | 入库单仓库 | 总部仓库 |
| 所属网点 | 数据归属网点 | 是 | 当前网点 | 浩瀚物流 |
| 使用网点 | 数据可见范围 | 是 | 全集团 | 全集团 |
| 当前状态 | 轮胎当前状态 | 是 | 未领用 | 未领用 |
| 当前车辆 | 如果已领用，记录车辆 | 否 | - | 京A12345 |
| 安装位置 | 如果已领用，记录位置 | 否 | - | 前左 |

**轮胎编号说明**：
- 使用轮胎出厂编号（通常在轮胎侧面，如DOT编号）
- 由用户手动录入
- 支持扫码枪快速录入
- 系统校验唯一性，不允许重复

**数据隔离说明**：
- **所属网点**：数据归属哪个网点，默认=当前登录用户所属网点
- **使用网点**：数据可见范围，默认=全集团（即所有网点都可以看到）
- 通过所属网点和使用网点实现数据隔离

### 2.3 轮胎状态

| 状态码 | 状态名称 | 说明 |
|-------|---------|------|
| 1 | 未领用 | 采购入库后的初始状态 |
| 2 | 已领用 | 已安装到车辆上 |
| 3 | 已回收 | 从车辆拆下，回收入库 |
| 4 | 已报废 | 生命周期结束 |
| 5 | 已售卖 | 已售出，生命周期结束 |

**状态流转规则**：

| 当前状态 | 可流转到 | 操作 |
|---------|---------|------|
| 未领用 | 已领用 | 领用 |
| 未领用 | 已售卖 | 售卖 |
| 已领用 | 未领用 | 撤销领用 |
| 已领用 | 已回收 | 回收 |
| 已回收 | 已领用 | 再次领用 |
| 已回收 | 已报废 | 报废 |
| 已回收 | 已售卖 | 售卖 |
| 已报废 | 已售卖 | 售卖 |
| 已售卖 | 原状态 | 撤销售卖（未创建财务单据） |

**状态流转图**：
```
采购入库 → 未领用 ──领用──→ 已领用 ──回收──→ 已回收
                ↑                ↓              ↓
                └────撤销领用─────┘          再次领用
                                                ↓
                                            已领用
                                                ↓
                                              回收
                                                ↓
                                            已回收
                                                ↓
                                              报废
                                                ↓
                                            已报废

未领用 ──售卖──→ 已售卖 ←──售卖── 已回收 ←──售卖── 已报废
   ↑                                                    ↓
   └────────────────────撤销售卖──────────────────────────┘
                    (仅已售卖且未创建财务单据)
```

**关键规则**：
1. 状态只能往下流转，除非撤销操作
2. 报废必须是"已回收"状态
3. 撤销领用仅支持"已领用"状态，按钮权限控制
4. 已报废、已售卖状态不可撤销领用
5. 撤销售卖仅支持"已售卖"状态且未创建财务单据

### 2.4 轮胎事件

记录轮胎生命周期中的关键事件：

| 事件类型 | 说明 | 是否影响库存 | 是否影响状态 |
|---------|------|------------|------------|
| 采购入库 | 轮胎采购入库 | 是（+1） | 未领用 |
| 领用 | 安装到车辆上 | 是（-1） | 已领用 |
| 回收 | 从车辆拆下入库 | 是（+1） | 已回收 |
| 撤销领用 | 撤销领用操作 | 是（+1） | 未领用 |
| 报废 | 轮胎报废 | 是（-1） | 已报废 |
| 售卖 | 轮胎售卖 | 是（-1） | 已售卖 |
| 撤销售卖 | 撤销售卖操作 | 是（+1） | 原状态 |
| 维修 | 轮胎维修 | 否 | 无变化 |
| 保养 | 轮胎保养 | 否 | 无变化 |
| 换位 | 轮胎换位 | 否 | 无变化 |
| 修补 | 轮胎修补 | 否 | 无变化 |

## 三、核心流程

### 3.1 采购入库流程（登记入库）

**触发点**：轮胎管理模块 → 轮胎卡片列表 → 「登记入库」按钮

**流程**：
1. 点击「登记入库」按钮，打开采购入库页面
2. 系统自动生成入库单号（如2603280001），用户可修改，校验唯一性
3. 填写基本信息：采购日期、供应商、所属仓库、所属网点、使用网点
4. 逐条添加轮胎明细：
   - **手动录入轮胎出厂编号**（支持扫码枪快速录入）
   - 填写品牌、规格、型号、采购价格
   - 物料单位固定为"条"，不可修改
5. 系统校验轮胎编号唯一性（不允许重复）
6. 保存后，系统为每条轮胎生成轮胎卡片，状态=未领用

**关键点**：
- 轮胎编号使用出厂编号，由用户手动录入
- 入库单号系统自动生成，用户可修改，校验唯一性
- 支持批量导入Excel（可选）
- 一次采购10条轮胎，生成10张独立的轮胎卡片
- 完全在轮胎管理模块操作，不依赖物料模块

### 3.2 领用流程

**触发点**：轮胎管理模块 → 轮胎领用

**流程**：
1. 选择车辆
2. 选择要领用的轮胎（可多选，仅显示"未领用"和"已回收"状态）
3. 选择安装位置（前左、前右、后左1、后右1、后左2、后右2、备胎）
4. 如果该车辆该位置已有轮胎，**强制处理旧轮胎**：
   - 选择"回收入库"或"直接报废"（不支持"暂不处理"）
   - 填写处理原因（必填）
   - 如果选择"直接报废"，必须上传报废照片
5. 确认领用
6. 新轮胎状态变为"已领用"，旧轮胎状态变为"已回收"或"已报废"

**关键点**：
- 支持一次领用多条轮胎
- 必须选择安装位置
- 领用时有旧轮胎必须处理，不支持"暂不处理"
- 可选关联维修单（事后也可在轮胎管理中手动关联）

### 3.3 中间过程（维修/保养/换位/修补）

**触发点**：轮胎管理模块 → 轮胎维护

**流程**：
1. 选择事件类型（维修/保养/换位/修补）
2. 选择轮胎编号（可多选）
3. 填写备注（可选）
4. 系统记录事件，不影响库存和状态

**关键点**：
- 完全灵活，想录就录，不想录不影响主流程
- 不改变轮胎状态
- 换位操作需要更新轮胎的安装位置

### 3.4 回收流程

**触发点**：轮胎管理模块 → 轮胎回收

**前置条件**：轮胎状态必须是"已领用"

**流程**：
1. 选择要回收的轮胎（可多选）
2. 选择回收仓库
3. 填写回收原因（必填）
4. 确认回收
5. 轮胎状态变为"已回收"，清空当前车辆和安装位置

**关键点**：
- 只能回收"已领用"状态的轮胎
- 回收原因必填
- 回收后的轮胎可以再次领用（循环使用）

### 3.5 撤销领用流程

**触发点**：轮胎管理模块 → 轮胎卡片详情 → 撤销领用

**前置条件**：轮胎状态必须是"已领用"，按钮权限控制

**流程**：
1. 点击"撤销领用"按钮
2. 填写撤销原因（必填）
3. 确认撤销
4. 轮胎状态变为"未领用"，清空车辆和安装位置

**关键点**：
- 仅"已领用"状态可撤销，已报废、已售卖不可撤销
- 按钮权限控制
- 撤销原因必填

### 3.6 报废流程

**触发点**：轮胎管理模块 → 轮胎报废

**前置条件**：轮胎状态必须是"已回收"

**流程**：
1. 选择要报废的轮胎（可多选）
2. 填写报废原因（必填）：磨损严重 / 损坏无法修复 / 老化 / 其他
3. 上传报废照片（必填）
4. 确认报废
5. 轮胎状态变为"已报废"

**关键点**：
- 必须是"已回收"状态才能报废（已领用的轮胎必须先回收再报废）
- 报废原因必填
- 报废照片必填
- 报废后不可恢复

### 3.7 报销流程

**触发点**：轮胎管理模块 → 轮胎报销

**前置条件**：
- 轮胎状态必须是"已报废"
- 未创建过报销单

**流程**：
1. 选择要报销的轮胎（可多选）
2. 系统自动生成报销单数据：报销金额=采购价格（不可修改）
3. 调用系统已有的财务单据创建流程，传入轮胎编号
4. 每个轮胎只能创建一次报销单

**关键点**：
- 每个轮胎只能创建一次报销单
- 报销金额=采购价格，系统自动填充
- 调用系统已有的财务单据创建流程，传入轮胎编号即可

### 3.8 售卖流程

**触发点**：轮胎管理模块 → 轮胎售卖

**适用状态**：未领用、已回收、已报废

**实际场景**：
- 新轮胎售卖创收（未领用状态）
- 回收轮胎售卖创收（已回收状态）
- 报废轮胎卖废品创收（已报废状态）

**流程**：
1. 选择要售卖的轮胎（可多选）
2. 每个轮胎单独填写售卖金额（必填）
3. 填写购买方（可选）和售卖日期（必填）
4. 确认售卖
5. 轮胎状态变为"已售卖"

**关键点**：
- 支持所有状态的轮胎售卖（未领用、已回收、已报废）
- 每个轮胎单独登记售卖金额
- 售卖后记录保留在系统中，不可删除
- 售卖后不可进行任何操作（除撤销售卖）

### 3.9 撤销售卖流程

**触发点**：轮胎管理模块 → 轮胎卡片详情 → 撤销售卖

**前置条件**：
- 轮胎状态必须是"已售卖"
- 未创建财务单据

**流程**：
1. 点击"撤销售卖"按钮
2. 填写撤销原因（必填）
3. 确认撤销
4. 轮胎状态恢复为售卖前的原状态
5. 清空售卖信息

**关键点**：
- 仅"已售卖"状态且未创建财务单据可撤销
- 已创建财务单据的不可撤销
- 撤销原因必填

## 五、数据分析

### 5.1 轮胎品牌使用情况

**分析维度**：
- 每个品牌使用数量
- 平均使用时长（采购→报废）
- 平均使用成本

**业务价值**：看哪个品牌更耐用，为采购决策提供数据支持

### 5.2 车辆轮胎消耗统计

**分析维度**：
- 每辆车领用了多少条轮胎
- 对应轮胎成本合计
- 平均每条轮胎使用时长

**业务价值**：看哪辆车轮胎成本最高，识别异常消耗车辆

### 5.3 轮胎履历表

**分析维度**：
- 按轮胎编号查询完整生命周期
- 采购 → 领用 ×N → 回收 ×N → 维修 → 报废

**业务价值**：追溯单条轮胎的完整历史

### 5.4 轮胎库存预警（可选）

**分析维度**：某个规格的轮胎库存低于安全库存时提醒

**业务价值**：避免缺货，及时采购

## 六、功能模块

### 6.1 轮胎卡片管理

**功能**：
- 轮胎卡片列表（含统计卡片：未领用/已领用/已回收/已报废/已售卖数量）

## 八、验收标准

### 8.1 功能验收

1. **登记入库**：入库单号自动生成且唯一，轮胎编号手动录入且唯一，保存后生成轮胎卡片
2. **领用**：领用后轮胎状态变为"已领用"，有旧轮胎时强制处理
3. **回收**：回收后轮胎状态变为"已回收"，可再次领用
4. **撤销领用**：仅"已领用"状态可撤销，按钮权限控制
5. **报废**：仅"已回收"状态可报废，原因和照片必填
6. **报销**：每个轮胎只能报销一次，金额=采购价格
7. **售卖**：支持所有状态售卖，每个轮胎单独登记金额，售卖后不可删除
8. **撤销售卖**：仅未创建财务单据可撤销
9. **统计分析**：能够输出品牌使用情况、车辆消耗统计、轮胎履历

### 8.2 性能验收

- 轮胎卡片列表查询响应时间 < 3秒
- 轮胎履历查询响应时间 < 2秒

### 8.3 数据隔离验收

- 所属网点数据隔离正确
- 使用网点可见范围正确

## 九、风险评估

### 9.1 技术风险

**风险1**：轮胎编号唯一性校验

**应对**：数据库唯一索引 + 前端实时校验

**风险2**：历史数据迁移

**应对**：新系统上线后，只管理新采购的轮胎，历史轮胎按需补录

### 9.2 业务风险

**风险3**：用户习惯改变

**应对**：提供用户培训，操作流程尽量简化

**风险4**：数据录入负担

**应对**：
- 支持扫码枪快速录入轮胎编号
- 支持批量导入Excel
- 中间过程（维修/保养）可选录入，不强制
